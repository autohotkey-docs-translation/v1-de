#ClipboardTimeout

Ändert, wie lange das Skript versuchen soll, auf die Zwischenablage zuzugreifen, falls der erste Versuch fehlschlägt.

#ClipboardTimeout Millisekunden

Parameter

Millisekunden

Die Länge des Intervalls in Millisekunden. Geben Sie -1 an, um den Versuch unendlich oft durchführen zu lassen. Geben Sie 0 an, um den Versuch nur einmal durchführen zu lassen. Skripte, die diese Direktive nicht enthalten, verwenden ein Zeitlimit von 1000 ms.

Bemerkungen

Einige Anwendungen halten die Zwischenablage über einen längeren Zeitraum offen, um zum Beispiel eine große Datenmenge zu schreiben oder zu lesen. Erhöhen Sie in solchen Fällen den Wert dieser Einstellung, um das Skript länger warten zu lassen, bevor es aufgibt und eine Fehlermeldung anzeigt.

Diese Einstellung gilt für alle Clipboard-Operationen, wozu auch die folgenden einfachen Beispiele gehören: Var := Clipboard und Clipboard := "Neuer Text".

Immer wenn das Skript auf die Verfügbarkeit der Zwischenablage wartet, können weder neue Threads noch Timer ausgeführt werden. Wenn der Benutzer allerdings einen Hotkey drückt, einen benutzerdefinierten Menüpunkt auswählt oder eine GUI-Aktion wie z. B. das Drücken einer Schaltfläche durchführt, wird dieses Ereignis gepuffert und zu einem späteren Zeitpunkt ausgelöst; das heißt, dass die Subroutine erst ausgeführt wird, wenn die Zwischenablage wieder verfügbar ist.

Diese Direktive bewirkt nicht, dass die Daten der Zwischenablage noch einmal gelesen werden, wenn der erste Versuch fehlschlägt. Vor [v1.1.16] war es so, dass diese Direktive das Skript bis zum Ablauf des Timeouts warten ließ, wodurch verhindert wurde, dass weitere Daten abgerufen werden konnten.
