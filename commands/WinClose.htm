WinClose

Schließt das angegebene Fenster.

WinClose [, Fenstertitel, Fenstertext, Wartezeit, Titelausnahme, Textausnahme]

Parameter

Fenstertitel

Ein Fenstertitel oder sonstige Kriterien, die das Zielfenster ermitteln sollen. Siehe Fenstertitel.

Fenstertext

In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (abrufbar mit Windows Spy). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.

Wartezeit

Ist dieser Parameter leer oder nicht vorhanden, wartet der Befehl überhaupt nicht. Ist eine 0 angegeben, wartet der Befehl 500 ms. Ansonsten wartet der Befehl die angegebene Anzahl an Sekunden (kann einen Dezimalpunkt oder einen Ausdruck enthalten), bis das Fenster geschlossen wird. Falls das Fenster innerhalb der Wartezeit nicht geschlossen wird, setzt das Script seine Ausführung fort. ErrorLevel wird von diesem Befehl nicht gesetzt - verwende daher IfWinExist oder WinWaitClose, um eindeutig festzustellen, ob ein Fenster geschlossen ist. Während der Befehl in einem Wartezustand ist, können neue Threads mittels Hotkey, benutzerdefinierte Menüeinträge oder Timer gestartet werden.

Titelausnahme

Ignoriert Fenster, deren Titel diesen Wert enthalten.

Textausnahme

Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Dieser Befehl sendet eine Close-Meldung ans Fenster. Das Ergebnis ist abhängig vom Fenster (es könnte dazu auffordern, Daten zu speichern, und so weiter).

Ist ein übereinstimmendes Fenster aktiv, wird vorrangig dieses Fenster geschlossen, anstatt ein anderes übereinstimmende Fenster darunter. Im Falle mehrerer übereinstimmender Fenster wird generell das oberste (aktuell verwendete) Fenster geschlossen.

Dieser Befehl bezieht sich immer auf das oberste übereinstimmende Fenster. Enthält Fenstertitel allerdings ahk_group Gruppenname, sind alle Fenster von der Gruppe betroffen.

WinClose sendet eine WM_CLOSE-Meldung ans Zielfenster, das eine etwas gewaltsame Methode zum Schließen des Fensters ist. Alternativ kann ein Fenster auch durch Senden der folgenden Nachricht geschlossen werden. Diese Vorgehensweise könnte ein anderes Verhalten erzeugen, weil sie das gleiche ist wie, als würde man ALT+F4 drücken oder den Schließen-Button in der Titelleiste anklicken:

PostMessage, 0x112, 0xF060,,, Fenstertitel, Fenstertext ; 0x112 = WM_SYSCOMMAND, 0xF060 = SC_CLOSE

Kann ein Fenster per WinClose nicht geschlossen werden, erzwinge das mit WinKill.

Fenstertitel und -texte unterscheiden zwischen Groß- und Kleinschreibung. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.
