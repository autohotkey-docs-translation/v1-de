If Var [not] in/contains Wert1,Wert2,...

Überprüft, ob der Inhalt der Variable mit einem der Elemente aus der Liste übereinstimmt.

if Var in Vergleichsliste
if Var not in Vergleichsliste

if Var contains Vergleichsliste
if Var not contains Vergleichsliste

Parameter

Var

Name der Variable, deren Inhalt überprüft werden soll. Bei dem Operator "in" muss die Variable exakt mit einem Element aus der Liste übereinstimmen. Bei dem Operator "contains" verhält es sich toleranter: Eine Übereinstimmung erfolgt, wann immer Var eines der Elemente als Teilstring enthält.

Vergleichsliste

Eine kommagetrennte Liste mit Strings, die mit dem Inhalt von Var verglichen werden sollen. Alle Leer- und Tabulatorzeichen an den angrenzenden Kommas sind von Bedeutung - das heißt, dass sie ein Teil des Übereinstimmungs-String sind. Wenn Vergleichsliste beispielsweise ABC , XYZ wäre, kann eine Übereinstimmung nur erfolgen, wenn Var ABC und ein Leerzeichen enthält, oder ein Leerzeichen und XYZ.

Zwei aufeinanderfolgende Kommas ergeben ein literales Komma. Zum Beispiel würde die folgende Zeile ein literales Komma nach dem Wort string1 erzeugen: If Var In string1,,,string2. Im Vergleich dazu würde die folgende Liste nur ein einzelnes Element mit einem literalen Komma dazwischen enthalten: If Var In einzelnes,,Element. Um ein leeres Element in die Liste einzufügen, muss das erste Zeichen ein Komma sein: If Var In ,string1,string2 (bei dem Operator "contains" wird ein leeres Element immer zur Übereinstimmung führen, weil ein leerer String in jedem String gefunden werden kann).

Da die Elemente in Vergleichsliste nicht als einzelne Parameter behandelt werden, kann die Liste komplett in einer Variable enthalten sein. Dies ist sowieso notwendig, wenn die Liste eine Länge von 16.383 Zeichen überschreitet (die maximale Länge einer Script-Zeile). Zum Beispiel könnte Vergleichsliste aus %Liste1%,%Liste2%,%Liste3% bestehen, von der jede Unterliste eine lange Liste mit Elementen enthält.

Bei jedem einzelnen Element in der Liste, das länger als 16384 Zeichen ist, werden die zusätzlichen Zeichen als neues Listenelement behandelt. Daher sollte man am besten vermeiden, solche Elemente überhaupt erst einzufügen.

Bemerkungen

Der Vergleich erfolgt immer alphabetisch, nicht numerisch. Zum Beispiel würde der String "11" nicht mit dem Listenelement "11.0" übereinstimmen.

Der Operator "contains" ist das gleiche wie, als würde man IfInString/IfNotInString verwenden, außer dass er mehrere Suchstrings unterstützt (von denen jeder eine Übereinstimmung bewirken kann).

Mit StringCaseSense On kann den Vergleich zeichengenau gemacht werden.

Wenn die Vergleichsliste zu lang ist, kann sie in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), um sie besser lesbar und verwaltbar zu machen.

Die Operatoren "between", "is", "in" und "contains" werden in Ausdrücken nicht unterstützt.

Siehe auch

if Var between, IfEqual/Greater/Less, IfInString, StringCaseSense, Blöcke, Else

Beispiele

if Var in exe,bat,com
    MsgBox Die Dateiendung ist eine ausführbare.

if Var in 2,3,5,7,11 ; Vermeide Leerzeichen in der Liste.
    MsgBox %Var% ist eine kleine Primzahl.

if Var contains 1,3  ; Beachte, dass hier die Werte als Strings verglichen werden, nicht als Zahlen.
    MsgBox Var enthält die Zahl 1 oder 3 (Var könnte auch eine 1, 3, 10, 21, 23, etc. sein)

if Var in %Vergleichsliste%
    MsgBox %Var% ist in der Liste.

InputBox, UserInput, Bitte JA oder NEIN eingeben
if UserInput not in ja,nein
    MsgBox Ihre Eingabe ist ungültig.
