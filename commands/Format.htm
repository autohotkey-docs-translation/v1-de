Format [v1.1.17+]

Formatiert eine variable Anzahl von Eingabewerten, basierend auf einen Formatstring.

String := Format(FormatStr [, Werte...])

Parameter

FormatStr

Ein Formatstring, bestehend aus literalem Text und Platzhalter, im Format {Index:Format}.

Index ist ein Integer, der angibt, welcher Eingabewert verwendet werden soll - dabei ist 1 der erste Wert.

Format ist eine optionale Formatspezifikation, wie unten beschrieben.

Lasse den Index weg, um den nächsten Eingabewert in der Sequenz zu verwenden (selbst wenn er bereits früher im String benutzt wurde). Zum Beispiel würde "{2:i} {:i}" den zweiten und dritten Eingabewert in einem dezimalen Integer formatieren, getrennt durch einem Leerzeichen. Fehlt Index, muss Format weiterhin ein : davor haben. Gib leere Klammern an, um den nächsten Eingabewert mit Standardformatierung zu verwenden: {}

Mit {{} und {}} können literale Klammern in den String eingefügt werden. Alle anderen ungültigen Platzhalter werden so wie sie sind in das Ergebnis eingefügt.

Leerraumzeichen innerhalb der Klammern sind nicht erlaubt (nur als Flag).

Werte

Eingabewerte, die formatiert und in den finalen String eingefügt werden sollen. Jeder Wert ist ein einzelner Parameter. Der erste Wert hat einen Index von 1.

Um einen Array mit Werten zu übergeben, muss ein variadischer Funktionsaufruf verwendet werden:

arr := [13, 240]
MsgBox % Format("{2:x}{1:02x}", arr*)

Formatspezifikationen

Jede Formatspezifikation kann die folgenden Komponente in dieser Reihenfolge enthalten (ohne den Leerzeichen):

Flag Breite .Präzision Typ

Flags dienen zum Ausrichten der Ausgabe und Ändern der Präfixe: - + 0 space #

Breite: Ein dezimaler Integer, der die Mindestbreite des formatierten Wertes bestimmt (in Zeichen). Standardmäßig werden Werte nach rechts ausgerichtet und Leerzeichen zum Auffüllen verwendet. Dies kann man mit den Flags - (links ausrichten) und 0 (Null-Präfix) überschreiben.

.Präzision: Ein dezimaler Integer, der je nach Ausgabetyp die maximale Anzahl von String-Zeichen, Dezimalstellen oder signifikante Stellen bestimmt. Es muss davor ein Dezimalpunkt sein. Die Angabe einer Präzision könnte dazu führen, dass der Wert gekürzt oder gerundet wird.

  • f, e, E: Präzision bestimmt die Anzahl von Ziffern nach dem Dezimalpunkt. Der Standardwert ist 6.
  • g, G: Präzision bestimmt die maximale Anzahl von signifikanten Stellen. Der Standardwert ist 6.
  • s: Präzision bestimmt die maximale Anzahl von Zeichen, die ausgegeben werden sollen. Alle darüberliegenden Zeichen werden nicht ausgegeben.
  • Bei den Integer-Typen (d, i, u, x, X, o) verhält sich Präzision wie Breite mit 0-Präfix und Standwert von 1.

Typ: Ein Zeichen, das kennzeichnet, wie der Eingabewert interpretiert werden soll. Fehlt diese Angabe, gilt standardmäßig s.

FlagBedeutungStandardwert
- Richtet das Ergebnis innerhalb der angegebenen Feldbreite nach links aus (und fügt bei Bedarf Leerzeichen in der rechten Seite ein). Rechtsausrichtung
+ Setzt ein Zeichen (+ oder -) vor dem Ausgabewert, sofern er ein vorzeichenbehafteter Typ ist. Zeichen erscheinen nur bei negativen vorzeichenbehafteten Werten (-).
0 Wenn Breite eine 0 als Präfix verwendet, werden solange Nullen davor gesetzt, bis die Mindestbreite erreicht wurde. Wenn sowohl 0 als auch - vorhanden sind, wird die 0 ignoriert. Wenn die 0 als Integer-Format (i, u, x, X, o, d) angegeben ist und eine Präzisionsspezifikation vorhanden ist - zum Beispiel {:04.d} - wird die 0 ignoriert. Kein Davorsetzen.
space Setzt ein einzelnes Leerzeichen vor dem Ausgabewert, sofern er vorzeichenbehaftet und positiv ist. Das Leerzeichen wird ignoriert, wenn sowohl space als auch + vorhanden sind. Es erscheinen keine Leerzeichen.
#

Setzt in Verbindung mit dem o-, x-, oder X-Format jeweils 0, 0x, oder 0X vor einem Ausgabewert ungleich Null.

In Verbindung mit dem e-, E-, f-, a- oder A-Format wird dieses Flag einen Dezimalpunkt im Ausgabewert erzwingen.

In Verbindung mit dem g- oder G-Format wird dieses Flag einen Dezimalpunkt im Ausgabewert erzwingen und das Kürzen von nachfolgenden Nullen verhindern.

In Verbindung mit dem c-, d-, i-, u-, oder s-Format wird es ignoriert.
TypzeichenArgumentAusgabeformat
d oder i Integer Vorzeichenbehafter dezimaler Integer.
u Integer Vorzeichenloser dezimaler Integer.
x oder X Integer Vorzeichenloser hexadezimaler Integer; verwendet "abcdef" oder "ABCDEF" je nach Groß- oder Kleinschreibung von x. Das 0x-Präfix ist erst enthalten, wenn das #-Flag vorhanden ist, wie z. B. in {:#x}. Um eine hexadezimale Formatierung wie bei SetFormat durchzuführen, muss man 0x{:x} oder ähnliches verwenden.
o Integer Vorzeichenloser oktaler Integer.
f Fließkomma Vorzeichenbehafteter Wert im Format [ - ]dddd.dddd - dabei sind dddd eine oder mehrere Dezimalstellen. Die Anzahl der Stellen vor dem Dezimalpunkt ist abhängig von der Größe der Nummer; die Anzahl der Stellen nach dem Dezimalpunkt ist abhängig von der angeforderten Präzision.
e Fließkomma Vorzeichenbehafteter Wert im Format [ - ]d.dddd e [Zeichen]dd[d] - dabei ist d eine Dezimalstelle, dddd eine oder mehrere Dezimalstellen, dd[d] zwei oder drei Dezimalstellen, basierend auf dem Ausgabeformat und Größe des Exponenten, und Zeichen + oder -.
E Fließkomma Das gleiche wie das e-Format, außer dass ein großes E den Exponenten kennzeichnet.
g Fließkomma Vorzeichenbehaftete Werte werden im f- oder e-Format angezeigt, abhängig davon, welcher von denen den angegebenen Wert kompakter darstellen können. Das e-Format wird nur verwendet, wenn der Exponent des Wertes kleiner als -4 oder größer gleich Präzision-Argument ist. Nachfolgende Nullen werden gekürzt; der Dezimalpunkt erscheint nur, wenn nach ihm eine oder mehrere Stellen folgen.
G Fließkomma Das gleiche wie das g-Format, außer dass ein großes E den Exponenten kennzeichnet (wo es notwendig ist).
a Fließkomma Vorzeichenbehafteter hexadezimaler Fließkommawert mit doppelter Präzision im Format [?]0xh.hhhh p±dd - dabei sind h.hhhh die Hexadezimalstellen (kleingeschrieben) der Mantisse, und dd eine oder mehrere Stellen für den Exponenten. Die Präzision bestimmt die Anzahl der Stellen nach dem Punkt.
A Fließkomma Das gleiche wie das a-Format, außer dass ein großes P den Exponenten kennzeichnet.
p Integer Zeigt das Argument als Speicheradresse in hexadezimalen Ziffern an.
s String Kennzeichnet einen String. Wenn der Eingabewert numerisch ist, wird er automatisch in einen String umgewandelt, mithilfe des aktuellen Zahlenformates des Scripts und bevor das Breite- und Präzision-Argument angewendet wird.
c Zeichencode Kennzeichnet ein einzelnes Zeichen als Ordinalwert, ähnlich wie Chr(n). Wenn der Eingabewert außerhalb des erwarteten Bereichs liegt, wird er umgekehrt.

Bemerkungen

Im Gegensatz zu printf werden Größenspezifikationen nicht unterstützt. Alle Integer- und Fließkomma-Eingabewerte sind 64-Bit.

Beispiel

; Einfache Substitution
s .= Format("{2}, {1}!`r`n", "Welt", "Hallo")
; Auffüllen mit Leerzeichen
s .= Format("|{:-10}|`r`n|{:10}|`r`n", "Links", "Rechts")
; Hexadezimal
s .= Format("{1:#x} {2:X} 0x{3:x}`r`n", 3735928559, 195948557, 0)
; Fließkomma
s .= Format("{1:0.3f} {1:.10f}", 4*atan(1))
