Finally [v1.1.14+]

Stellt sicher, dass immer, wenn eine Try-Anweisung erfolgt, eine oder mehrere Anweisungen (Befehle oder Ausdrücke) ausgeführt werden.

Finally Anweisung
Finally
{
    Anweisungen
}

Bemerkungen

Jedes Finally muss mit einer Try- oder Catch-Anweisung darüber verbunden werden. Ein Finally bezieht sich immer auf die nächste noch nicht beanspruchte Try-Anweisung darüber, sofern dieses Verhalten nicht mit einem Block geändert wurde.

Das Verhalten einer Try-Anweisung ist abhängig davon, ob catch oder finally vorhanden ist. Weitere Informationen findest du unter Try.

Goto, Break, Continue und Return können nicht verwendet werden, um einen Finally-Block zu verlassen, weil es sonst notwendig wäre, jegliche Ablaufsteuerung innerhalb eines Try-Blocks zu unterdrücken. Wenn Try beispielsweise return 42 verwendet, würde der Wert 42 erst zurückgegeben werden, nachdem der Finally-Block ausgeführt wurde. Der Versuch, mittels einem dieser Befehle aus einem Finally-Block zu springen, wird beim Laden des Scripts als Fehler erkannt, sofern möglich, oder ansonsten während das Script ausgeführt wird.

Vor v1.1.19.02 existierte ein Bug, der Ablaufsteuerungen daran hinderte, innerhalb von Try zu funktionieren, während Finally vorhanden war Return wurde fälschlicherweise in Finally zugelassen, aber ignoriert, wenn eine Ausnahme ausgelöst wurde.

Je nach Bedarf kann der "One True Brace Style" bei der Finally-Anweisung angewendet werden. Zum Beispiel:

try {
    ...
} finally {
    ...
}

try {
    ...
} catch e {
    ...
} finally {
    ...
}

Beispiele

try
{
    ToolTip, Bearbeite ...
    Beispiel1()
}
catch e
{
    ; Um mehr über das Objekt zu erfahren, das in e enthalten ist, siehe Catch.
    MsgBox, 16,, % "Ausnahme ausgelöst!`n`nWas: " e.what "`nDatei: " e.file
        . "`nZeile: " e.line "`nNachricht: " e.message "`nZusatz: " e.extra
}
finally
{
    ToolTip ; Versteckt das ToolTip
}

MsgBox, Fertig!

; Diese Funktion hat einen Finally-Block, der als Aufräumcode fungiert.
Beispiel1()
{
    try
        Beispiel2()
    finally
        MsgBox, Dies wird immer ausgeführt, unabhängig von Ausnahmen
}

; Diese Funktion wird fehlschlagen, wenn die Minuten ungerade sind.
Beispiel2()
{
    if Mod(A_Min, 2)
        throw Exception("Test-Ausnahme")
    MsgBox, Beispiel2 ist nicht fehlgeschlagen
}
