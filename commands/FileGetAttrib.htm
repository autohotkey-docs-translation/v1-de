FileGetAttrib

Ermittelt, ob eine Datei zum Beispiel schreibgeschützt oder versteckt ist.

FileGetAttrib, AusgabeVar [, Dateiname]
Attributsstring := FileExist(Dateimuster)

Parameter

AusgabeVar Name der Variable, in der der ermittelte Text gespeichert werden soll.
Dateiname Name der Datei, die sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist. Ist dieser Parameter nicht vorhanden, wird stattdessen die aktuelle Datei von der innersten umschließenden Loop (Dateien & Ordner) verwendet.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Die interne Variable A_LastError enthält den Rückgabewert der Systemfunktion GetLastError().

Bemerkungen

Der zurückgegebene String enthält Buchstaben aus dem String "RASHNDOCT":

R = READONLY (schreibgeschützt)
A = ARCHIVE (archiviert)
S = SYSTEM (Systemdatei)
H = HIDDEN (versteckt)
N = NORMAL
D = DIRECTORY (Verzeichnis)
O = OFFLINE
C = COMPRESSED (komprimiert)
T = TEMPORARY (temporär)


Mit dem folgenden Beispiel kann festgestellt werden, ob ein bestimmtes Attribut im String vorkommt:

FileGetAttrib, Attribute, C:\Eigene Datei.txt
IfInString, Attribute, H
    MsgBox Die Datei ist versteckt.

Weiterhin ist es möglich, den 8.3-Kurzname einer Datei wie folgt abzurufen:

Loop, C:\Eigene Dokumente\Adressenliste.txt
    Kurzpfadname = %A_LoopFileShortPath%  ; Ergebnis: C:\EIGENE~1\ADRESS~1.TXT

Mit einem ähnlichen Verfahren kann der lange Name eines 8.3-Kurznamen ermittelt werden.

Verwandte Befehle

FileExist(), FileSetAttrib, FileGetTime, FileSetTime, FileGetSize, FileGetVersion, Loop (Dateien & Ordner)

Beispiel

FileGetAttrib, AusgabeVar, C:\Neuer Ordner
