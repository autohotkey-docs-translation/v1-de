Text

Beschreibung: Ein Bereich mit randlosem Text, den der Benutzer nicht bearbeiten kann. Oft verwendet, um andere Steuerelemente zu bezeichnen.

Beispiel: Gui, Add, Text,, Bitte Name eingeben:

In diesem Fall ist der letzte Parameter der String, welcher angezeigt werden soll. Der String kann LF-Zeichen (`n) enthalten, um neue Zeilen zu beginnen. Außerdem kann eine lange Zeile in mehreren kurzen Zeilen mit einem Fortsetzungsbereich aufgetrennt werden.

Wenn eine Breite (W) in den Optionen angegeben ist, aber keine Zeilen (R) oder Höhe (H), dann erfolgt wie vorgesehen automatischer Zeilenumbruch und die Höhe des Steuerelements wird automatisch gesetzt.

Da der Inhalt des Steuerelements im letzten Parameter des GUI-Befehls ist, müssen direkte Kommas nicht geschützt werden. Das gilt auch für den letzten Parameter von allen anderen Befehlen.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird das Label EigeneSubroutine automatisch ausgeführt, sobald der Benutzer auf den Text klickt. Beispielsweise kann damit ein unterstrichener, blauer Hyperlink simuliert werden:

Gui, Font, underline
Gui, Add, Text, cBlue gLaunchGoogle, Hier klicken, um Google zu starten.
Gui, Font, norm
Gui, Show
Return

LaunchGoogle:
Run www.google.com
Return

Ein Doppelklick kann erkannt werden, wenn überprüft wird, ob A_GuiEvent das Wort DoubleClick enthält.

Das kaufmännische Und (&) kann im Text verwendet werden, um einen Buchstaben zu unterstreichen. Zum Beispiel:

Gui, Add, Text,, &First Name:
Gui, Add, Edit

Im obigen Beispiel wird der Buchstabe F unterstrichen, d. h. der Benutzer kann die Tastenkombination Alt+F benutzen, um den Tastaturfokus auf das erste eingabefähige Steuerelement zu setzen, welches nach dem Text-Steuerelement hinzugefügt wurde. Um stattdessen ein kaufmännisches Und direkt anzuzeigen, verwendet zwei aufeinanderfolgende kaufmännische Und-Zeichen (&&). Um die spezielle Behandlung der kaufmännischen Und-Zeichen zu deaktivieren, fügt 0x80 in den Optionen des Steuerelements ein.

Siehe allgemeine Optionen für weitere Optionen (z. B. Right, Center und Hidden). Siehe auch: Position und Größe von Steuerelementen anpassen.

Edit

Beschreibung: Ein Bereich, der mit freiem Text vom Benutzer beschrieben werden kann.

Beispiel: Gui, Add, Edit, r9 vEigenesEdit, Text erscheint innerhalb des Edit-Steuerelements (Lasst diesen Parameter weg, um anfangs leer zu sein).

Das Steuerelement wird mehrzeilig, falls der Text mehrere Zeilen hat. Mit r3 in den Optionen wird zum Beispiel ein dreizeiliges Edit-Steuerelement mit den folgenden Standardeigenschaften erstellt: eine vertikale Bildlaufleiste, automatischer Zeilenumbruch und Abfangen der Entertaste bei der Eingabe anstatt Auslösen der Standard-Schaltfläche im Fenster.

Um eine neue Zeile in einem mehrzeiligen Edit-Steuerelement zu beginnen, fügt im letzten Parameter (Inhalte) ein LF-Zeichen (`n) oder das Paar CR+LF (`r`n) ein. Beide Methoden erzeugen direkt vorhandene CR+LF-Paare (`r`n) innerhalb des Edit-Steuerelements. Wenn jedoch das Steuerelement mittels Gui Submit oder GuiControlGet in die dazugehörige Variable gespeichert wird, dann wird immer jedes CR+LF-Paar (`r`n) in ein LF-Zeichen (`n) umgewandelt. Um Text in eine Datei zu speichern, befolgt dieses Beispiel: FileAppend, %EigenesEdit%, C:\Saved File.txt

Wenn im Steuerelement automatischer Zeilenumbruch aktiviert ist (Standard bei mehrzeiligen Edit-Steuerelementen), dann erzeugt jeglicher Umbruch, der bei der Benutzereingabe entsteht, keine LF-Zeichen (nur die Entertaste ist dazu fähig).

Mehrzeilige Edit-Steuerelemente in Windows 95/98/Me sind auf 64 KB Text begrenzt, ab Windows NT/2k/XP jedoch auf ca. 4 GB. Sobald der Benutzer Text eingibt, wird mehr Speicher nach Bedarf zugewiesen.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird das Label EigeneSubroutine automatisch ausgeführt, sobald der Inhalt des Steuerelements geändert wird.

TIPP: Um eine Textdatei in ein Edit-Steuerelement zu laden, verwendet FileRead und GuiControl. Zum Beispiel:

Gui, Add, Edit, R20 vEigenesEdit
FileRead, FileContents, C:\Eigene Datei.txt
GuiControl,, EigenesEdit, %FileContents%

Edit-Optionen (um eine Option zu entfernen, setzt ein Minuszeichen davor):

Limit: Beschränkt die Eingabe des Benutzers auf die Breite des sichtbaren Eingabefelds. Alternativ kann auch direkt danach eine Zahl angegeben werden, um die Eingabe auf eine bestimmte Anzahl an Zeichen zu begrenzen. Mit Limit10 beispielsweise können nicht mehr als 10 Zeichen eingegeben werden.

Lowercase: Die eingegebenen Zeichen vom Benutzer werden automatisch in Kleinbuchstaben umgewandelt.

Multi: Ermöglicht mehrere Textzeilen. Allerdings ist die Angabe normalerweise nicht nötig, weil es automatisch erkannt wird, basierend auf Höhe (H), Zeilen (R) oder Inhalt (Text).

Number: Verhindert, dass der Benutzer etwas anderes als Zahlen in das Feld eingeben kann (allerdings ist es noch möglich, Nicht-Zahlen einzufügen). Als alternativer Weg kann die numerische Eingabe mit dem UpDown-Steuerelement erzwungen werden.

Password: Versteckt die Eingabe des Benutzers (wie bei einer Passworteingabe) durch das Ersetzen der Zeichen mit dem Maskierungszeichen. Wenn ein benutzerdefiniertes Maskierungszeichen gewünscht ist, fügt es direkt nach dem Wort Password an. Password* beispielsweise ersetzt den schwarzen Kreis (standardmäßig in Windows XP) mit einem Sternchen. Hinweis: Diese Option hat keine Auswirkung auf mehrzeilige Edit-Steuerelemente.

ReadOnly: Verhindert, dass der Benutzer den Inhalt des Steuerelements ändern kann. Allerdings kann der Text noch gescrollt, markiert und in die Zwischenablage kopiert werden.

Tn: Mit dem Buchstaben T können Tabstopps innerhalb eines mehrzeiligen Edit-Steuerelements gesetzt werden (da Tabstopps die Spaltenpositionen der springenden direkten TAB-Zeichen abrufen, können sie dazu verwendet werden, den Text in Spalten zu formatieren). Wenn der Buchstabe T nicht verwendet wird, dann werden die Tabstopps alle 32 DLU (Dialogeinheit) gesetzt (die Breite jeder DLU wird vom Betriebssystem bestimmt). Wenn der Buchstabe T nur einmal verwendet wird, dann werden die Tabstopps auf je n Einheiten über die gesamte Breite des Steuerelements verteilt. Mit Gui, Add, Edit, vEigenesEdit r16 t64 beispielsweise wird die Standard-Distanz zwischen den Tabstopps verdoppelt. Um benutzerdefinierte Tabstopps zusetzen, verwendet den Buchstaben T wie im folgenden Beispiel mehrfach: Gui, Add, Edit, vEigenesEdit r16 t8 t16 t32 t64 t128. Ein Tabstopp ist jeweils eine absolute Spaltenposition in der Liste, bis zu einem Maximum von 50 Tabstopps. Hinweis: Tabstopps erfordern ein mehrzeiliges Edit-Steuerelement.

Uppercase: Die eingegebenen Zeichen vom Benutzer werden automatisch in Großbuchstaben umgewandelt.

WantCtrlA [v1.0.44+]: Mit -WantCtrlA wird der Benutzer daran gehindert, den ganzen Text mit Steuerung+A im Edit-Steuerelement zu markieren.

WantReturn: Mit -WantReturn erkennt ein mehrzeiliges Edit-Steuerelement nicht die Entertaste. Das Drücken der Entertaste ist dann das gleiche wie das Drücken der Standard-Schaltfläche im Fenster (falls vorhanden). In diesem Fall kann der Benutzer Steuerung+Enter benutzen, um eine neue Zeile zu beginnen.

WantTab: Bei einer Tabulatoreingabe wird ein Tabulator-Zeichen erzeugt und nicht zum nächsten Steuerelement navigiert. Ohne diese Option kann der Benutzer die Tastenkombination Steuerung+Tab verwenden, um ein Tab-Zeichen in einem mehrzeiligen Edit-Steuerelement zu erzeugen. Hinweis: Obwohl WantTab auch in einem einzeiligen Edit-Steuerelement funktioniert, werden Tab-Zeichen als ein leeres Box-Zeichen angezeigt (gelten aber weiterhin als wahre Tab-Zeichen).

-Wrap: Deaktiviert automatischen Zeilenumbruch in einem mehrzeiligen Edit-Steuerelement. Da dieser Style nicht geändert werden kann, nachdem das Steuerelement erstellt wurde, verwendet eine folgende Methode: 1) Destroy verwenden, danach Fenster und deren Steuerelemente neu erstellen; oder 2) Zwei übereinander liegende Edit-Steuerelemente erstellen, eins mit aktiviertem Zeilenumbruch, das andere ohne. Der zurzeit nicht verwendet wird, bleibt leer und/oder versteckt.

Siehe allgemeine Optionen für weitere Optionen (z. B. Right, Center und Hidden). Siehe auch: Position und Größe von Steuerelementen anpassen.

Ein besseres Edit-Steuerelement: HiEdit ist ein freies Edit-Steuerelement mit mehreren Tabs und für große Dateien, welches sehr wenig Speicher verbraucht. Es kann sowohl Text- als auch Binärdateien bearbeiten. Für Details siehe www.autohotkey.com/forum/topic19141.html

UpDown

Beschreibung: Ein Paar Pfeiltasten, die der Benutzer verwenden kann, um einen Wert zu erhöhen oder zu verringern. Standardmäßig fügt sich ein UpDown-Steuerelement automatisch an das zuvor hinzugefügte Steuerelement an. Dieses vorherige Steuerelement ist auch als Buddy-Steuerelement bekannt. Das bekannteste Beispiel dafür ist der "Spinner" (Drehrad), ein UpDown-Steuerelement, verbunden mit einem Edit-Steuerelement. Zum Beispiel:

Gui, Add, Edit
Gui, Add, UpDown, vEigenesUpDown Range1-10, 5

Im obigen Beispiel ist das Edit-Steuerelement das Buddy-Steuerelement von UpDown. Sobald der Benutzer eine Pfeiltaste verwendet, wird die Zahl im Edit-Steuerelement automatisch erhöht oder verringert.

Ein Buddy-Steuerelement von UpDown kann auch ein Text- oder ListBox-Steuerelement sein. Aufgrund von Beschränkungen des Betriebssystems funktionieren andere Steuerelemente (z. B. ComboBox und DropDownList) nicht richtig mit g-Labels und andere Funktionen.

Die Anfangsposition des UpDown-Steuerelements kann im letzten Parameter angegeben werden (wenn nicht vorhanden, dann bei 0 oder bei der Zahl im zulässigen Bereich, die am nächsten an der Null ist).

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) die numerische Position des UpDown-Steuerelements abgerufen. Wenn das UpDown-Steuerelement mit einem Edit-Steuerelement verbunden ist und der Benutzer die Eingabe nicht überprüfen will, dann ist es am besten, den Wert des UpDown-Steuerelements zu verwenden. Denn das UpDown-Steuerelement übergibt immer eine Zahl in einem gültigen Bereich, selbst wenn der Benutzer keine Zahl oder eine Zahl außerhalb des Bereichs im Edit-Steuerelement eingegeben hat. Zusätzlich werden Zahlen mit mehr als 3 Stellen standardmäßig mit einem Tausender-Trennzeichen (z. B. Komma) ausgegeben. Diese Trennzeichen werden aber nur in der Ausgabevariable des Edit-Steuerelements gespeichert.

Wenn das UpDown-Steuerelement ein g-Label hat, dann wird es ausgeführt, sobald der Benutzer eine Pfeil-Schaltfläche oder Pfeiltaste auf der Tastatur drückt. Jeder Start von g-Label speichert auch die Position des UpDown-Steuerelements in die zugewiesene Ausgangsvariable (falls vorhanden).

UpDown-Optionen:

Horz: Ermöglicht links/rechts Pfeiltasten, anstatt oben/unten. Standardmäßig isoliert Horz auch das Steuerelement (kein Buddy). Das kann mit Horz 16 in den Optionen des Steuerelements überschrieben werden.

Left: Setzt das UpDown-Steuerelement an die linke Seite des Buddy Control, anstatt an die rechte Seite.

Range: Bestimmt einen benutzerdefinierten Bereich (standardmäßig von 0 bis 100). Nach dem Wort Range erfolgt der Minimalwert, ein Bindestrich und der Maximalwert. Zum Beispiel: Range1-1000 ermöglicht eine Zahl zwischen 1 und 1000, Range-50-50 eine Zahl zwischen -50 und 50 und Range-10--5 eine Zahl zwischen -10 und -5. Die Minimal- und Maximalwerte können vertauscht werden, um die Pfeile in die entgegengesetzte Richtung zu verschieben. Der breiteste zulässige Bereich ist von -2147483648 bis 2147483647. Allerdings benötigen Windows 95 und NT4 Internet Explorer 5.0 oder höher, um einen breiteren Bereich als -32.767 bis 32.767 zu unterstützen. Wenn das Buddy-Steuerelement eine ListBox ist, dann wird ein Standardbereich von 32767 bis 0 für vertikale und das Umgekehrte für horizontale Steuerelemente (Horz) verwendet.

Wrap: Das Steuerelement springt auf das entgegengesetzte Ende des Bereichs, wenn der Minimal- oder Maximalwert überschritten wird. Ohne Wrap stoppt das Steuerelement, wenn der Minimal- oder Maximalwert erreicht ist.

-16: Ein vertikales UpDown-Steuerelement wird isoliert, d. h. es hat kein Buddy Control. Somit berücksichtigt das Steuerelement auch jede angegebene Breite, Höhe und Position, anstatt sich nach der Größe des Buddy-Steuerelements zu richten. Darüber hinaus merkt sich ein isoliertes UpDown-Steuerelement intern die eigene Position. Diese Position kann beispielsweise mit Gui Submit abgerufen werden.

0x80: Mit 0x80 in den Optionen werden die Tausender-Trennzeichen weggelassen, die normalerweise jeweils zwischen drei Dezimalstellen im Buddy-Steuerelement vorkommen. Dieser Style wird normalerweise nicht verwendet, weil die Trennzeichen von den Nummern weggelassen werden, sobald es vom UpDown-Steuerelement selbst abgerufen wird (statt vom Buddy-Steuerelement).

Beliebige Schrittweite: In diesem Script von NumEric wird veranschaulicht, wie die Schrittweite des UpDown-Steuerelements geändert werden kann (z. B. 5 oder 0.1).

Siehe auch: Position und Größe von Steuerelementen anpassen.

Picture (oder Pic)

Beschreibung: Ein Bereich mit einem Bild (siehe die letzten beiden Absätze für unterstützte Dateitypen). Der letzte Parameter enthält den Dateinamen des Bildes, das sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist.

Beispiel: Gui, Add, Picture, w300 h-1, C:\Eigene Bilder\Company Logo.gif

Um die aktuelle Breite und/oder Höhe des Bildes zu erhalten, lasst die Optionen W und/oder H weg. Ansonsten wird das Bild auf die angegebene Breite und/oder Höhe skaliert (diese Breite und Höhe bestimmt auch, welches Icon von einer .ICO-Datei mit mehreren Icons geladen wird). Um das Bild unter Beibehaltung des Seitenverhältnisses zu verkleinern oder zu vergrößern, verwendet -1 für eine Abmessung und eine positive Zahl für die andere Abmessung. Mit "w200 h-1" beispielsweise wird die Breite des Bildes auf 200 Pixel gesetzt und die Höhe automatisch eingestellt. Wenn das Bild nicht geladen oder angezeigt werden kann (z. B. Datei nicht gefunden), wird das Steuerelement leer gelassen und die Breite und Höhe auf Null gesetzt.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer auf das Bild klickt. Ein Doppelklick kann erkannt werden, wenn überprüft wird, ob A_GuiEvent das Wort DoubleClick enthält.

Um das Bild als Hintergrund für andere Steuerelemente zu verwenden, fügt es davor ein. Wenn allerdings diese Steuerelemente eingabefähig sind und das Bild ein g-Label hat, dann erstellt das Bild erst nach diesen Steuerelementen und fügt 0x4000000 (WS_CLIPSIBLINGS) zu den Optionen des Bildes hinzu. Mit diesem Trick kann das Bild auch der Hintergrund eines Tab- oder ListView-Steuerelements sein.

Icon, Cursor und animierte Cursor: Icon und Cursor können von folgenden Dateitypen geladen werden: ICO, CUR, ANI, EXE, DLL, CPL, SCR und andere Typen, die Icon-Ressourcen enthalten. Um eine beliebige Icon-Gruppe in einer Datei zu verwenden, fügt in den Optionen das Wort Icon ein, gefolgt von der Gruppennummer. Im folgenden Beispiel wird das Standard-Icon aus der zweiten Icon-Gruppe verwendet: Gui, Add, Picture, Icon2, C:\Eigene Anwendung.exe

Mit AltSubmit in den Optionen wird die Datei GDIPlus.dll von Microsoft verwendet, um das Bild laden, welches zu unterschiedlichen Ergebnissen bei der Anzeige von GIF-, BMP- und Icon-Bilder führt. Damit wird beispielsweise ein ICO/GIF-Bild mit transparentem Hintergrund zu einer transparenten Bitmap, bei der die Option BackgroundTrans in Effekt treten kann. Wenn GDIPlus nicht verfügbar ist (siehe nächster Absatz), dann wird AltSubmit ignoriert und das Bild mit der normalen Methode geladen.

Alle Betriebssysteme unterstützen GIF, JPG, BMP, ICO, CUR und ANI-Bilder. In Windows XP (oder höher) werden zusätzliche Bildformate wie PNG, TIF, Exif, WMF und EMF unterstützt. Betriebssysteme älter als XP können unterstützt werden, wenn die freie GDIPlus.dll von Microsoft in den Ordner der AutoHotkey.exe kopiert wird (im Falle eines kompilierten Scripts muss die DLL-Datei in den Ordner des Scripts). Um die DLL herunterzuladen, sucht in www.microsoft.com mit folgenden Begriffen: gdi redistributable

Aufgrund eines Fehlers in Windows 95/98/Me verursachen animierte Cursor, die größer als 90x90 sind, einen Absturz des Scripts.

Animierte GIF-Bilder: Animierte GIF-Dateien können auch in einem Bild-Steuerelement angezeigt werden, sie werden jedoch nicht animiert. Zur Lösung dieses Problems kann die AniGIF.dll verwendet werden (die frei ist für nicht-kommerzielle Nutzung), siehe www.autohotkey.com/forum/topic19264.html

Button

Beschreibung: Eine Schaltfläche, die gedrückt werden kann, um eine Aktion auszulösen. In diesem Fall ist der letzte Parameter der Name der Schaltfläche (wird auf der Schaltfläche selbst angezeigt), wobei der Name LF-Zeichen (`n) enthalten kann, um neue Zeilen zu beginnen.

Beispiel: Gui, Add, Button, Default, OK

Das obige Beispiel enthält das Wort Default in den Optionen, damit wird "OK" zur Standard-Schaltfläche. Die Aktion der Standard-Schaltfläche wird automatisch ausgelöst, sobald der Benutzer die Entertaste drückt, es sei denn, der Tastatur-Fokus liegt auf einer anderen Schaltfläche oder ein mehrzeiliges Edit-Steuerelement hat den WantReturn-Style. Um später die Standard-Schaltfläche zu wechseln, siehe folgendes Beispiel: GuiControl, +default, Cancel. Damit das Fenster später keine Standard-Schaltfläche hat, siehe folgendes Beispiel: GuiControl, -default, OK

Das kaufmännische Und (&) kann im Namen verwendet werden, um einen Buchstaben zu unterstreichen. Zum Beispiel:

Gui, Add, Button,, &Pause

Im obigen Beispiel wird der Buchstabe P unterstrichen und ermöglicht dem Benutzer, Alt+P als Tastenkombination zu verwenden. Um stattdessen ein kaufmännisches Und direkt anzuzeigen, verwendet zwei aufeinanderfolgende kaufmännische Und-Zeichen (&&).

Wenn der Schaltfläche ein eindeutiges g-Label fehlt, dann wird automatisch ein Label zugewiesen. Wenn zum Beispiel das erste GUI-Fenster eine OK-Schaltfläche enthält, dann wird das ButtonOK-Label (falls vorhanden) ausgeführt, wenn die Schaltfläche gedrückt wird. Label von anderen GUI-Fenstern mit Ausnahme des ersten GUI's beinhalten zusätzlich die Fensternummer, zum Beispiel: 2ButtonOK.

Wenn der Text auf der Schaltfläche Leerzeichen oder andere Zeichen (&`r`n`t`) enthält, dann werden automatisch diese Zeichen weggelassen. Zum Beispiel hat die Schaltfläche mit der Bezeichnung "&Pause" automatisch das Label ButtonPause. Außerdem hat die Schaltfläche "Save && Exit" das Label ButtonSaveExit (die zwei Und-Zeichen werden verwendet, um ein kaufmännisches Und direkt darzustellen).

Bekannte Einschränkung: Bei bestimmten Desktop-Designs wird der Text in den Schaltflächen nicht richtig angezeigt. Wenn dies geschieht, versucht es mit der Option -Wrap. Allerdings wird dadurch auch verhindert, dass der Text mehrzeilig ist.

Checkbox

Beschreibung: Ein kleines Kästchen, das markiert oder nicht markiert werden kann, um An/Aus, Ja/Nein, usw. zu repräsentieren.

Beispiel: Gui, Add, Checkbox, vShipToBillingAddress, An Rechnungsadresse senden?

Der letzte Parameter ist die Bezeichnung neben dem Kästchen, die typischerweise als Aufforderung oder als Beschreibung dient. Der String kann LF-Zeichen (`n) enthalten, um neue Zeilen zu beginnen. Wenn eine Breite (W) in den Optionen angegeben ist, aber keine Zeilen (R) oder Höhe (H), dann erfolgt wie Vorgesehen automatischer Zeilenumbruch und die Höhe des Steuerelements wird automatisch gesetzt. Die zugewiesene Ausgabevariable der CheckBox (falls vorhanden) erhält die Nummer 1 für markiert, 0 für nicht markiert, und -1 für grau/unbestimmt.

Mit dem Wort Check3 in den Optionen wird der dritte Status aktiviert und ein graues anstelle eines schwarzen Häckchens angezeigt (der graue Zustand zeigt an, dass die CheckBox weder markiert noch nicht markiert ist). Mit dem Wort Checked oder CheckedGray in den Optionen hat die CheckBox anfangs jeweils ein schwarzes oder graues Häkchen. Nach dem Wort Checked kann auch eine 0, 1 oder -1 für den Anfangsstatus erfolgen. Das heißt, dass "Checked" und "Checked%VarMitEins%" identisch sind.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer auf die CheckBox klickt oder es ändert.

Bekannte Einschränkung: Bei bestimmten Desktop-Designs wird der Text in den Schaltflächen nicht richtig angezeigt. Wenn dies geschieht, versucht es mit der Option -Wrap. Allerdings wird dadurch auch verhindert, dass der Text mehrzeilig ist.

Radio

Die Radio-Schaltfläche ist ein kleiner leerer Kreis, dass markiert (An) oder nicht markiert (Aus) werden kann.

Beispiel: Gui, Add, Radio, vEigeneRadioGruppe, Auf alle Artikel im Lager warten, bevor sie abgeschickt werden.

Diese Steuerelemente befinden sich normalerweise in Radio-Gruppen, welche 2 oder mehr Radio-Schaltflächen enthalten. Wenn der Benutzer auf eine Radio-Schaltfläche klickt, um es zu aktivieren, dann werden alle anderen Radio-Schaltflächen in der Gruppe automatisch deaktiviert (der Benutzer kann auch innerhalb einer Gruppe mit den Pfeiltasten navigieren). Eine Radio-Gruppe wird automatisch um alle fortlaufend hinzugefügten Radio-Schaltflächen erstellt. Um eine neue Gruppe zu beginnen, verwendet das Wort Group in den Optionen der ersten Schaltfläche der neuen Gruppe - oder fügt einfach ein anderes Steuerelement ein, dass automatisch eine neue Gruppe startet.

Im letzten Parameter kann die Bezeichnung der Radio-Schaltfläche angegeben werden. Diese Bezeichnung wird typischerweise als Aufforderung oder als Beschreibung verwendet und kann LF-Zeichen (`n) enthalten, um neue Zeilen zu beginnen. Wenn eine Breite (W) in den Optionen angegeben ist, aber keine Zeilen (R) oder Höhe (H), dann erfolgt wie Vorgesehen automatischer Zeilenumbruch und die Höhe des Steuerelements wird automatisch gesetzt.

Mit dem Wort Checked in den Optionen wird die Schaltfläche anfangs aktiviert. Nach dem Wort Checked kann auch eine 0 oder 1 für den Anfangsstatus erfolgen: 0 für markiert und 1 für nicht markiert. Das heißt, dass "Checked" und "Checked%VarMitEins%" identisch sind.

Die zugewiesene Ausgabevariable der Radio-Schaltfläche (falls vorhanden) erhält die Nummer 1 für markiert und 0 für nicht markiert. Wenn jedoch nur eine Schaltfläche in einer Radio-Gruppe eine Variable hat, dann ruft diese Variable stattdessen wird die Nummer der aktuell ausgewählten Schaltfläche ab: 1 ist die erste Radio-Schaltfläche (nach ursprünglicher Erstellungsreihenfolge), 2 die Zweite und so weiter. Wenn keine Schaltfläche markiert ist, wird 0 gespeichert.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer die Schaltfläche aktiviert. Im Gegensatz zum Modus mit einer Variable muss das g-Label für jede Schaltfläche in einer Radio-Gruppe festgelegt werden. Damit wird Flexibilität gewährleistet, Klicks von bestimmten Schaltflächen zu ignorieren. Ein Doppelklick kann erkannt werden, wenn überprüft wird, ob A_GuiEvent das Wort DoubleClick enthält.

Bekannte Einschränkung: Bei bestimmten Desktop-Designs wird der Text in den Schaltflächen nicht richtig angezeigt. Wenn dies geschieht, versucht es mit der Option -Wrap. Allerdings wird dadurch auch verhindert, dass der Text mehrzeilig ist.

DropDownList (oder DDL)

Beschreibung: Eine Auswahlliste, die angezeigt wird, wenn eine kleine Schaltfläche gedrückt wird. In diesem Fall ist der letzte Parameter eine Auswahlliste, deren Einträge durch senkrechte Striche getrennt werden (z. B. Auswahl1|Auswahl2 |Auswahl3).

Beispiel: Gui, Add, DropDownList, vColorChoice, Schwarz|Weiß|Rot|Grün|Blau

Mit zwei senkrechten Strichen danach wird ein Eintrag vorausgewählt (z. B. Rot|Grün||Blau). Alternativ kann mit dem Wort Choose in den Optionen, unmittelbar gefolgt von der Nummer, ein Eintrag sofort vorausgewählt werden. Mit Choose5 beispielsweise wird der fünfte Eintrag vorausgewählt (wie bei anderen Optionen, kann es auch eine Variable wie zum Beispiel Choose%Var% sein). Nachdem das Steuerelement erstellt wurde, kann mit GuiControl die Auswahl geändert oder Einträge in der Liste hinzugefügt/entfernt werden.

Mit dem Wort Uppercase oder Lowercase in den Optionen werden alle Einträge in der Liste automatisch in Groß-oder Kleinbuchstaben umgewandelt. Mit dem Wort Sort wird der Inhalt der Liste automatisch alphabetisch sortiert (hat auch Auswirkung auf später hinzugefügte Einträge mittels GuiControl). Diese Option ermöglicht auch inkrementelle Suche, sobald die Liste aufgeklappt wird; damit wird ein Eintrag ausgewählt, wenn die ersten Zeichen des Namens eingegeben werden.

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) der Text des zurzeit ausgewählten Eintrags abgerufen. Wenn jedoch das Steuerelement die AltSubmit-Eigenschaft hat, dann wird mit der Ausgabevariable stattdessen die Positionsnummer des Eintrags abgerufen (der erste Eintrag ist 1, der zweite 2 usw.).

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer einen neuen Eintrag auswählt.

Mit der R- oder H-Option kann die Höhe der Aufklappliste kontrolliert werden. Mit R5 beispielsweise ist die Liste 5 Zeilen hoch. Wenn sowohl R als auch H weggelassen werden, dann wird die Liste automatisch erweitert, um die Vorteile der verfügbaren Desktophöhe des Benutzers zu übernehmen (allerdings zeigen Betriebssysteme älter als Windows XP standardmäßig 3 Zeilen).

Das Trennzeichen zwischen den Feldern muss nicht ein senkrechter Strich sein. Mit Gui +Delimiter`n beispielsweise ist das Trennzeichen ein LF und mit Gui +DelimiterTab ein Tabulator (`t).

ComboBox

Beschreibung: Das Gleiche wie DropDownList, aber mit dem Unterschied, dass Text als Alternative zur Auswahl eines Eintrags aus der Liste eingegeben werden kann.

Beispiel: Gui, Add, ComboBox, vColorChoice, Rot|Grün|Blau|Schwarz|Weiß

Zusätzlich zu den gleichen Optionen wie bei DropDownList, kann mit dem Wort Limit in den Optionen die Eingabe des Benutzers auf die sichtbare Breite des Edit-Steuerelements eingeschränkt werden. Mit dem Wort Simple verhält sich die ComboBox wie ein Edit-Steuerelement mit einer ListBox darunter.

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) der Text des zurzeit ausgewählten Eintrags abgerufen. Wenn jedoch das Steuerelement die AltSubmit-Eigenschaft hat, dann wird mit der Ausgabevariable stattdessen die Positionsnummer des Eintrags abgerufen (der erste Eintrag ist 1, der zweite 2 usw.). In jedem Fall enthält die Ausgabevariable den Inhalt des Edit-Steuerelements der ComboBox, wenn kein Eintrag ausgewählt ist.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer einen neuen Eintrag auswählt.

ListBox

Beschreibung: Ein relativ hohes Feld mit einer Auswahlliste. In diesem Fall ist der letzte Parameter eine Auswahlliste, deren Einträge durch senkrechte Striche getrennt werden (z. B. Auswahl1|Auswahl2 |Auswahl3).

Beispiel: Gui, Add, ListBox, vColorChoice, Rot|Grün|Blau|Schwarz|Weiß

Mit zwei senkrechten Strichen danach wird ein Eintrag (oder auch mehrere Einträge) vorausgewählt (es wird die Multi-Option benötigt, um mehrere Einträge auszuwählen). Alternativ kann mit dem Wort Choose in den Optionen, unmittelbar gefolgt von der Nummer, ein Eintrag sofort vorausgewählt werden. Mit Choose5 beispielsweise wird der fünfte Eintrag vorausgewählt. Nachdem das Steuerelement erstellt wurde, kann mit GuiControl die Auswahl geändert oder Einträge in der Liste hinzugefügt/entfernt werden.

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) der Text des zurzeit ausgewählten Eintrags abgerufen. Wenn jedoch das Steuerelement die AltSubmit-Eigenschaft hat, dann wird mit der Ausgabevariable stattdessen die Positionsnummer des Eintrags abgerufen (der erste Eintrag ist 1, der zweite 2 usw.).

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer einen neuen Eintrag auswählt. Sobald der Benutzer auf einen Eintrag doppelt klickt, enthält die interne Variable A_GuiEvent den String DoubleClick anstelle von Normal. Außerdem enthält die Variable A_EventInfo die Position des Eintrags, dass doppelt geklickt wurde (1 ist der erste Eintrag, 2 der Zweite und so weiter).

Wenn eine hohe Anzahl an Einträgen hinzugefügt wird, dann kann die Performance verbessert werden, indem GuiControl, -Redraw, EigeneListBox vor der Operation und GuiControl, +Redraw, EigeneListBox nach der Operation verwendet werden.

ListBox-Optionen:

Choose: Siehe oben.

Multi: Damit können mehrere Einträge mittels Umschalt+Klick und Steuerung+Klick gleichzeitig ausgewählt werden (mit der Nummer 8 anstelle dem Wort Multi können Einträge ohne Umschalt/Steuerung+Klick markiert werden). In diesem Fall speichert Gui Submit eine Liste mit Einträgen in die Ausgangsvariable des Steuerelements, die durch senkrechte Striche getrennt werden. Wenn jedoch die Option AltSubmit aktiv ist, dann speichert Gui Submit stattdessen eine Liste mit Eintragsnummern, die durch senkrechte Striche getrennt werden. 1|2|3 beispielsweise bedeutet, dass die ersten drei Einträge ausgewählt sind. Um einzelne Einträge zu extrahieren, verwendet eine Parsing-Schleife wie in diesem Beispiel:

Loop, parse, EigeneListBox, |
{
    MsgBox Auswahlnummer %A_Index% ist %A_LoopField%.
}

Das Trennzeichen zwischen den Feldern muss nicht ein senkrechter Strich sein. Mit Gui +Delimiter`n beispielsweise ist das Trennzeichen ein LF und mit Gui +DelimiterTab ein Tabulator (`t).

ReadOnly: Verhindert, dass die Einträge sichtbar hervorgehoben werden, wenn sie markiert werden (aber Gui Submit speichert weiterhin den ausgewählten Eintrag).

Sort: Mit dem Wort Sort wird der Inhalt der Liste automatisch alphabetisch sortiert (hat auch Auswirkung auf später hinzugefügte Einträge mittels GuiControl). Diese Option ermöglicht auch inkrementelle Suche; damit wird ein Eintrag ausgewählt, wenn die ersten Zeichen des Namens eingegeben werden.

Tn: Mit dem Buchstaben T können Tabstopps gesetzt werden, um den Text in Spalten zu formatieren. Wenn der Buchstabe T nicht verwendet wird, dann werden die Tabstopps alle 32 DLU (Dialogeinheit) gesetzt (die Breite jeder DLU wird vom Betriebssystem bestimmt). Wenn der Buchstabe T nur einmal verwendet wird, dann werden die Tabstopps auf je n Einheiten über die gesamte Breite des Steuerelements verteilt. Mit "Gui, Add, ListBox, vEigeneListBox t64" beispielsweise wird die Standard-Distanz zwischen den Tabstopps verdoppelt. Um benutzerdefinierte Tabstopps zusetzen, verwendet den Buchstaben T wie im folgenden Beispiel mehrfach: Gui, Add, ListBox, vEigeneListBox t8 t16 t32 t64 t128. Ein Tabstopp ist jeweils eine absolute Spaltenposition in der Liste, bis zu einem Maximum von 50 Tabstopps.

0x100: Mit 0x100 in den Optionen wird der Style LBS_NOINTEGRALHEIGHT aktiviert. Die ListBox wird dazu gezwungen, exakt die angegebene Höhe zu verwenden, anstatt eine Höhe, die das Anzeigen einer Zeile am unteren Rand unterdrückt. Diese Option verhindert auch, dass die ListBox schrumpft, wenn die Schrift geändert wird.

Um die Anzahl der Textzeilen zu bestimmen (oder die Höhe und Breite), siehe Position und Größe von Steuerelementen anpassen.

Hotkey

Beschreibung: Sieht aus wie ein einzeiliges Edit-Steuerelement, aber mit dem Unterschied, dass eingegebene Tastenkombinationen vom Benutzer erkannt werden. Wenn der Benutzer zum Beispiel Steuerung+Alt+C auf einer deutschen Tastaturbelegung drückt, dann erscheint im Feld "STRG + ALT + C".

Beispiel: Gui, Add, Hotkey, vChosenHotkey

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) die Modifikatoren und der Name des Hotkeys abgerufen, die mit dem Hotkey-Befehl kompatibel sind. Beispiele: ^!C, +!Home, +^Down, ^Numpad1, !NumpadEnd. Wenn kein Hotkey im Steuerelement ist, dann ist die Ausgabevariable leer. Hinweis: Einige Tasten werden gleich angezeigt, selbst sie als verschiedene Namen abgerufen werden. Zum Beispiel werden sowohl ^Numpad7 wie auch ^NumpadHome als "STRG + 7 (ZEHNERTASTATUR)" angezeigt.

Standardmäßig startet das Steuerelement ohne angegebenen Hotkey. Um stattdessen einen Standardwert anzugeben, verwendet Modifikatoren und den Namen im letzten Parameter wie in diesem Beispiel: Gui, Add, Hotkey, vChosenHotkey, ^!p
Es werden nur die Modifikatoren ^ (Steuerung), ! und Umschalt (+) werden unterstützt. Siehe Keyboard, Maus und Joystickreferenz für weitere Tasten.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer den Hotkey ändert. Jeder Start von g-Label speichert auch den Hotkey in die zugewiesene Ausgangsvariable (falls vorhanden). Hinweis: g-Label wird auch ausgeführt, wenn ein unvollständiger Hotkey vorhanden ist. Wenn der Benutzer beispielsweise Strg-Taste gedrückt hält, dann wird g-Label einmal gestartet und die Ausgabevariable enthält nur einen Zirkumflex (^). Wenn der Benutzer die Tastenkombination abgeschlossen hat, dann wird das g-Label wieder gestartet und die Variable enthält den kompletten Hotkey.

Um die Eingabe von Hotkey-Typen einzuschränken, fügt nach dem Wort Limit die Summe folgender Zahlen an:

1: Verhindert unmodifizierte Tasten
2: Verhindert Umschalt-Tasten
4: Verhindert Steuerungstasten
8: Verhindert Alt-Tasten
16: Verhindert Umschalt-Strg-Tasten
32: Verhindert Umschalt-Alt-Tasten
64: Dieser Wert wird nicht unterstützt (und wird sich nicht korrekt verhalten).
128: Verhindert Umschalt-Steuerung-Alt-Tasten.

Mit Limit1 beispielsweise können unmodifizierte Hotkeys wie Buchstaben und Zahlen nicht eingegeben werden und mit Limit15 werden mindestens zwei Modifikatorentasten benötigt. Wenn der Benutzer eine verbotene Modifikator-Kombination verwendet, dann erscheint Steuerung+Alt automatisch und wird sichtbar ersetzt.

Das Hotkey-Steuerelement hat nur begrenzte Möglichkeiten. Zum Beispiel werden nicht Maus/Joystick-Hotkeys oder die Windows-Taste (LWin und RWin) unterstützt. Als Übergangslösung können beliebig viele Checkbox-Steuerelemente verwendet werden, die zusätzliche Modifikatoren wie die Win-Taste anbieten.

DateTime

Beschreibung: Sieht wie ein einzeiliges Edit-Steuerelement aus, das jedoch nur die Eingabe von Datum und/oder Zeit akzeptiert. Ein ausklappbarer Kalender ist ebenfalls vorhanden.

Beispiel: Gui, Add, DateTime, vEigenesDateTime, LongDate

Der letzte Parameter kann Folgendes enthalten:

(nicht vorhanden): Wenn weggelassen, dann wird das kurze Datumsformat verwendet, abhängig vom aktuellen Gebietsschema des Benutzers. Könnte zum Beispiel wie folgt aussehen: 17.04.2011

LongDate: Verwendet das lange Datumsformat, abhängig vom aktuellen Gebietsschema des Benutzers. Könnte zum Beispiel wie folgt aussehen: 17.04.2011

Time: Zeigt nur die Zeit an, abhängig vom aktuellen Gebietsschema des Benutzers. Obwohl das Datum nicht angezeigt wird, ist es weiterhin im Steuerelement und kann zusammen mit der Zeit im YYYYMMDDHH24MISS-Format abgerufen werden.

(benutzerdefiniertes Format): Verwendet eine beliebige Kombination der Datum- und Zeitformate. d.M.yy HH:mm wird beispielsweise zu 17.4.11 12:19. Und dddd MMMM d, yyyy hh:mm:ss beispielsweise zu Sonntag April 17, 2011 12:22:45. Damit Buchstaben und Zahlen direkt angezeigt werden können, umschließt sie wie im diesem Beispiel mit Hochkommas: 'Datum:' MM/dd/yy 'Zeit:' hh:mm:ss tt Nicht-alphanumerische Zeichen wie Leerzeichen, Tabulatoren, Schrägstriche, Doppelpunkte, Kommas und andere Satzzeichen müssen nicht von Hochkommas umschlossen werden. Die Ausnahme ist das Hochkomma selbst: Zur Darstellung dieses Zeichens müssen vier aufeinanderfolgende Hochkommas ('''') verwendet werden, oder nur zwei, wenn das Hochkomma sich bereits innerhalb von zwei äußeren Hochkommas befindet.

Verwendung von DateTime

Um zu Beginn ein anderes Datum als heute zu verwenden, fügt in den Optionen das Wort Choose ein, gefolgt von einem Datum im YYYYMMDD-Format. Mit Choose20110417 beispielsweise wird 17. April 2011 vorausgewählt (wie bei anderen Optionen, kann es auch eine Variable wie zum Beispiel Choose%Var% sein). Damit kein Datum/Uhrzeit ausgewählt ist, verwendet ChooseNone. ChooseNone erstellt außerdem eine Checkbox innerhalb des Steuerelements, die deaktiviert ist, sobald das Steuerelement kein Datum hat. Sobald das Steuerelement kein Datum hat, ermittelt Gui Submit und GuiControlGet einen leeren Wert (leerer String).

Die Tageszeit kann optional vorhanden sein. Erfolgt aber immer nach dem Datum, wenn das Format ausgewählt oder abgerufen wird. Das Format der Zeitangabe ist HH24MISS (Stunden, Minuten, Sekunden), wo HH24 im 24-Stunden-Format zum Ausdruck kommt. Somit ist ein vollständiger Datum-Zeit-String im YYYYMMDDHH24MISS-Format.

Wenn ein Datum im YYYYMMDDHH24MISS-Format angegeben wird, dann ist nur der führende Teil notwendig. Alle nachfolgende Elemente, die weggelassen werden, haben folgende Standardwerte:
MM: Monat 01
DD: Tag 01
HH24: Stunde 00
MI: Minute 00
SS: Sekunde 00

In dem ausklappbaren Kalender kann der Heute-String unten angeklickt werden, um das heutige Datum auszuwählen. Darüber hinaus ist das Jahr und der Monatsname anklickbar und ermöglichen eine einfache Navigation zu einem neuen Monat oder Jahr.

Navigation mit der Tastatur: Mit Up/Down (Pfeiltasten), NumpadPlus/Minus und Home/End können die Werte des Steuerelements erhöht oder verringert werden. Mit Left und Right (Pfeiltasten) kann von Feld zu Feld innerhalb des Steuerelements navigiert werden. Innerhalb des ausklappbaren Kalenders kann mit den Pfeiltasten von Tag zu Tag navigiert werden; mit PageUp/Down einen Monat zurück/vor; mit Strg+PageUp/Down ein Jahr zurück/vor; und mit Home/End jeweils den ersten/letzten Tag im Monat auswählen.

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) das Datum und die Uhrzeit im YYYYMMDDHH24MISS-Format abgerufen. Sowohl das Datum als auch die Uhrzeit sind vorhanden, unabhängig davon, ob sie tatsächlich im Steuerelement sichtbar sind.

Das g-Label (falls vorhanden) wird ausgeführt, sobald der Benutzer das Datum oder die Uhrzeit ändert. Bei jedem Start wird die zugewiesenen Ausgabevariable (falls vorhanden) des Steuerelements mit dem aktuell ausgewählten Datum/Uhrzeit aktualisiert.

Windows 95 und NT4 benötigen mindestens Internet Explorer 3.0, damit DateTime-Steuerelemente unterstützt werden.

DateTime-Optionen:

Choose: Siehe oben.

Range: Beschränkt die Datumsauswahl auf einem bestimmten Zeitraum. Nach dem Wort Range erfolgt das minimale und maximale Datum im YYYYMMDD-Format (mit einem Bindestrich dazwischen). Mit Range20050101-20050615 beispielsweise wird das Datum auf die ersten 5.5 Monaten des Jahres 2005 beschränkt. Das Minimum oder Maximum kann weggelassen werden, damit das Steuerelement in dieser Richtung unbegrenzt ist. Beispiele: Mit Range20010101 kann kein Tag vor 2001 ausgewählt werden und mit Range-20091231 (führender Bindestrich) kann kein Tag nach 2009 ausgewählt werden. Ohne Range kann jedes Datum zwischen den Jahren 1601 und 9999 ausgewählt werden. Die Uhrzeit kann nicht eingeschränkt werden.

Right: Bewirkt, dass der ausklappbare Kalender auf der rechten anstelle der linken Seite des Steuerelements aufgeklappt wird.

1: Mit der Nummer 1 in den Optionen wird der ausklappbare Monatskalender mit einem UpDown-Steuerelement auf der rechten Seite des Steuerelements ersetzt, um Datum-Zeit-Werte zu modifizieren.

2: Mit der Nummer 2 in den Optionen wird eine CheckBox innerhalb des Steuerelements unterstützt, dass der Benutzer deaktivieren kann, um kein Datum/Uhrzeit auszuwählen. Nachdem das Steuerelement erstellt wurde, kann diese Option nicht mehr geändert werden.

MonthCal

Beschreibung: Ein hohes und breites Steuerelement, das alle Tage des Monats im Kalender-Format anzeigt. Der Benutzer kann ein einzelnes Datum oder einen Datumsbereich auswählen.

Beispiel: Gui, Add, MonthCal, vEigenesKalender

Damit ein anderes Datum als heute vorgewählt ist, fügt im letzten Parameter ein Datum im YYYYMMDD-Format ein (z. B. 20050531). Ein Datumsbereich kann auch vorgewählt werden, indem ein Bindestrich zwischen zwei Daten eingefügt wird (z. B. 20050525-20050531).

Es ist normalerweise am besten, Breite (W) und Höhe (H) im MonthCal-Steuerlement wegzulassen, weil die Größe automatisch auf einem Monat angepasst wird. Um mehrere Monate vertikal darzustellen, verwendet R2 oder höher in den Optionen. Um mehrere Monate horizontal darzustellen, verwendet W-2 oder höher in den Optionen. Diese Optionen können gleichzeitig vorhanden sein, um in beide Richtungen zu erweitern.

Der Heute-String unterhalb des Steuerelements kann angeklickt werden, um das heutige Datum auszuwählen. Darüber hinaus ist das Jahr und der Monatsname anklickbar, dass eine einfache Auswahl eines neuen Monats oder Jahrs ermöglicht.

Im Gegensatz zum ausklappbaren Kalender von DateTime wird die Tastaturnavigation in einem MonthCal-Steuerelement in der Regel nicht unterstützt.

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) das ausgewählte Datum im YYYYMMDD-Format (ohne Uhrzeit) abgerufen. Wenn allerdings die Mehrfachauswahl-Option aktiv ist, dann werden die minimalen und maximalen Daten mit einem Bindestrich dazwischen ermittelt (z. B. 20050101-20050108). Wenn nur ein einzelnes Datum in einem Kalender mit Mehrfachauswahl ausgewählt wurde, dann ist das Minimum und Maximum vorhanden, aber identisch. Mit StringSplit können die Daten getrennt werden. Mit dem folgendem Beispiel wird das Minimum in Date1 und das Maximum in Date2 gespeichert: StringSplit, Date, EigenesMonthCal, -

Wenn das MonthCal-Steuerelement einen g-Label hat, dann wird die zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) mit dem aktuell ausgewählten Datum oder Bereich aktualisiert. Standardmäßig kann das Label nur aufgerufen werden, wenn: 1) der Benutzer die Auswahl ändert, oder 2) nur alle zwei Minuten, falls ein neuer Tag beginnt (dieses Verhalten ist eine Eigenart des Betriebssystems). Wenn allerdings das Wort AltSubmit in den Optionen des Steuerelements ist, dann wird g-Label öfters ausgeführt und die interne Variable A_GuiEvent enthält das Wort "Normal" für eine Änderung des Datums, die Nummer 1 für das Anklicken des Datums und die Nummer 2, wenn das MonthCal-Steuerelement die "Mauserfassung" loslässt. Wenn zum Beispiel der Benutzer auf neues Datum doppelklickt, dann wird das Label fünfmal ausgeführt: Einmal mit Normal, zweimal mit 1 und zweimal mit 2. Das kann verwendet werden, um die Doppelklicks durch Messen der Zeit zwischen den Instanzen der Nummer 1 zu erkennen.

Wenn das Datum im YYYYMMDD-Format angegeben wird, dann kann der MM- und DD-Teil weggelassen werden, welche dann als 1 angesehen werden. Zum Beispiel wird 200205 als 20020501, sowie 2005 als 20050101 angesehen.

Windows 95 und NT4 benötigen mindestens Internet Explorer 3.0, damit MonthCal-Steuerelemente unterstützt werden.

MonthCal-Optionen:

Multi: Mehrfachauswahl. Erlaubt dem Benutzer, mit Umschalt+Klick oder Klicken und Ziehen eine Reihe von benachbarten Daten auszuwählen (der Benutzer kann weiterhin ein einzelnes Datum auswählen). Diese Option kann direkt angegeben werden oder automatisch in Kraft treten, wenn ein Auswahlbereich bei der Erstellung des Steuerelements angegeben wird. Zum Beispiel: Gui, Add, MonthCal, vEigenesKal, 20050101-20050108. Nachdem das Steuerelement erstellt wurde, kann diese Option nicht mehr geändert werden.

Range: Beschränkt die Datumsauswahl auf einem bestimmten Zeitraum. Nach dem Wort Range erfolgt das minimale und maximale Datum im YYYYMMDD-Format (mit einem Bindestrich dazwischen). Mit Range20050101-20050615 beispielsweise wird die Auswahl auf die ersten 5.5 Monaten des Jahres 2005 beschränkt. Das Minimum oder Maximum kann weggelassen werden, damit das Steuerelement in dieser Richtung unbegrenzt ist. Beispiele: Mit Range20010101 kann kein Tag vor 2001 ausgewählt werden und mit Range-20091231 (führender Bindestrich) kann kein Tag nach 2009 ausgewählt werden. Ohne Range kann jedes Datum zwischen den Jahren 1601 und 9999 ausgewählt werden.

4: Mit der Nummer 4 in den Optionen werden die Wochennummern (1-52) an der linken Seite jeder Reihe angezeigt. Woche 1 wird als erste Woche definiert, die mindestens vier Tage enthält.

8: Mit der Nummer 8 in den Optionen wird verhindert, dass das Datum von Heute im Steuerelement umkreist wird.

16: Mit der Nummer 16 in den Optionen wird verhindert, dass das Datum von Heute im Steuerelement unten angezeigt wird.

Slider

Beschreibung: Ein Schieberegler, dass der Benutzer entlang einer vertikalen oder horizontalen Strecke bewegen kann. Die Standard-Lautstärkeregelung in der Taskleiste ist ein Beispiel für einen Schieberegler.

Beispiel: Gui, Add, Slider, vEigenesSlider, 50

Die Startposition des Schiebereglers kann im letzten Parameter angegeben werden. Wenn der letzte Parameter weggelassen wird, dann beginnt der Schieberegler bei 0 oder bei der Zahl im zulässigen Bereich, die am nächsten an der Null ist.

Der Benutzer kann den Regler wie folgt verschieben: 1) mit der Maus ziehen, 2) auf den Bereich der Schiene klicken, 3) das Mausrad drehen, während das Steuerelement den Fokus hat, oder 4) folgende Tasten drücken, während das Steuerelement den Fokus hat: Pfeiltasten, Bild-Auf, Bild-Ab, Pos1 und Ende.

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) die numerische Position des Slider-Steuerelements abgerufen. Die Position wird auch in die Ausgabevariable gespeichert, sobald das g-Label des Steuerelements ausgeführt wird.

Wenn der Schieberegler ein g-Label hat, dann wird es standardmäßig nur ausgeführt, wenn der Benutzer aufgehört hat, den Regler zu bewegen (z. B. durch Loslassen der Maustaste, nachdem es gezogen wurde). Wenn allerdings das Wort AltSubmit in den Optionen des Steuerelements ist, dann wird das g-Label bei allen Ereignissen des Schiebereglers ausgeführt und die interne Variable A_GuiEvent enthält eine folgende Zahl oder einen folgenden String:

0: Der Benutzer betätigt die linke oder obere Pfeiltaste.
1: Der Benutzer betätigt die rechte oder untere Pfeiltaste.
2: Der Benutzer betätigt die Bild-Auf-Taste.
3: Der Benutzer betätigt die Bild-Ab-Taste.
4: Der Benutzer bewegt den Schieberegler mit dem Mausrad oder beendet das Ziehen und Ablegen an eine neue Position.
5: Der Benutzer zieht den Schieberegler zurzeit mit der Maus, d. h. die Maustaste wird derzeit gedrückt.
6: Der Benutzer betätigt die Pos1-Taste, um den Regler ganz nach links oder nach oben zu verschieben.
7: Der Benutzer betätigt die Ende-Taste, um den Regler ganz nach rechts oder nach unten zu verschieben.
Normal: Der Benutzer ist mit dem Verschieben des Reglers fertig, entweder mit der Maus oder Tastatur. Hinweis: Mit Ausnahme der Mausradbewegung (#4) wird das g-Label auch dann wieder bei dem Normal-Ereignis ausgeführt, wenn es bereits bei einem Zahl-Ereignis von oben ausgeführt wurde.

Slider-Optionen:

Buddy1 und Buddy2: Bestimmt bis zu zwei vorhandene Steuerelemente, die automatisch an den Enden des Schiebereglers positioniert werden. Buddy1 wird an der linken oder oberen Seite angezeigt (je nachdem, ob die Option Vertical vorhanden ist). Buddy2 wird an der rechten oder unteren Seite angezeigt. Nach dem Wort Buddy1 oder Buddy2 kann der Variablenname eines vorhandenen Steuerelements angegeben werden. Mit Buddy1EigenerObertext wird beispielsweise das Steuerelement mit dem Namen EigenerObertext zugewiesen. Windows 95 und NT4 benötigen Internet Explorer 3.0 oder höher, um diese Option zu unterstützen.

Center: Der Einstellknopf (welcher durch den Benutzer verschoben wird) ist an beiden Enden stumpf, anstatt an einem Ende spitz.

Invert: Dreht das Steuerelement um, damit der niedrigere Wert rechts/unten ist, anstatt links/oben. Das wird normalerweise benutzt, um einen vertikalen Schieberegler als eine traditionelle Lautstärkeregelung zu verwenden. Hinweis: Die unten beschriebene ToolTip-Option berücksichtigt nicht die die Umkehrung und sollte in diesem Fall nicht verwendet werden.

Left: Der Einstellknopf (welcher durch den Benutzer verschoben wird) ist am oberen Ende spitz. Aber wenn die Vertical-Option aktiviert ist, dann ist der Einstellknopf an der rechten Seite spitz.

Line: Bestimmt die Anzahl der zu bewegenden Positionen, wenn der Benutzer eine Pfeiltaste betätigt. Nach dem Wort Line kann die Anzahl der zu bewegenden Positionen angegeben werden. Zum Beispiel: Line2

NoTicks: Die Skalenstriche entlang der Strecke werden weggelassen.

Page: Bestimmt die Anzahl der zu bewegenden Positionen, wenn der Benutzer die Bild-Auf- oder Bild-Ab-Taste betätigt. Nach dem Wort Page kann die Anzahl der zu bewegenden Positionen angegeben werden. Zum Beispiel: Page10

Range: Bestimmt einen benutzerdefinierten Bereich (standardmäßig von 0 bis 100). Nach dem Wort Range erfolgt der Minimalwert, ein Bindestrich und der Maximalwert. Zum Beispiel: Range1-1000 ermöglicht eine Zahl zwischen 1 und 1000, Range-50-50 eine Zahl zwischen -50 und 50 und Range-10--5 eine Zahl zwischen -10 und -5.

Thick: Bestimmt die Länge des Einstellknopfs (welcher durch den Benutzer verschoben wird). Nach dem Wort Thick kann die Dicke in Pixel angegeben werden (z. B. Thick30). Um die Dicke in Windows XP oder höher zu ändern, ist es wahrscheinlich notwendig, die Center-Option zu verwenden oder das Design des Steuerelements zu entfernen (dazu muss -Theme in den Optionen des Steuerelements angegeben werden).

TickInterval: Ermöglicht Skalenstriche entlang der Strecke in einem bestimmten Intervall. Nach dem Wort TickInterval kann das Intervall angegeben werden, um zusätzliche Skalenstriche anzuzeigen (ist das Intervall nicht vorhanden, wird standardmäßig 1 verwendet). TickInterval10 beispielsweise zeigt alle 10 Positionen einen Skalenstrich.

ToolTip: Erstellt ein Tooltip, der die numerische Position des Schiebereglers anzeigt, wenn er vom Benutzer gezogen wird. Um den Tooltip an einer anderen Position anzuzeigen, verwendet Folgendes: ToolTipLeft oder ToolTipRight (für horizontale Schieberegler); ToolTipTop oder ToolTipBottom (für vertikale Schieberegler). Windows 95 und NT4 benötigen Internet Explorer 3.0 oder höher, um diese Option zu unterstützen.

Vertical: Damit kann der Schieberegler nach oben oder nach unten verschoben werden, anstatt von links nach rechts.

Die oben genannten Optionen können mittels GuiControl geändert werden, nachdem das Steuerelement erstellt wurde.

Progress

Beschreibung: Ein zweifarbiger Balken, der anzeigt, wie viel Fortschritt einer Operation gemacht wurde.

Beispiel: Gui, Add, Progress, w300 h20 cBlue vEigenesProgress

Die Anfangsposition des Balkens kann im letzten Parameter angegeben werden (wenn nicht vorhanden, dann bei 0 oder bei der Zahl im zulässigen Bereich, die am nächsten an der Null ist). Um die Position des Balkens später zu ändern, befolgt diese Beispiele, die sich auf einen Fortschrittsbalken mit dem zugewiesenem Variablennamen EigenesProgress beziehen:

GuiControl,, EigenesProgress, +20  ; Die Position wird um 20 erhöht.
GuiControl,, EigenesProgress, 50  ; Die Position wird auf 50 gesetzt.

Bei horizontalen Fortschrittsbalken wird die Dicke des Balkens durch die Höhe des Steuerelements bestimmt. Bei vertikalen Fortschrittsbalken durch die Breite des Steuerelements.

Progress-Optionen:

Cn: Ändert die Farbe des Balkens. Ersetzt n mit einer HTML-Primärfarbe oder einem 6-stelligen RGB-Farbwert. Beispiele: cRed, cFFFF33, cDefault. Wenn die C-Option nicht verwendet wird (oder cDefault angegeben ist), dann wird die Standardfarbe des Betriebssystems verwendet.

BackgroundN: Ändert die Hintergrundfarbe des Balkens. Ersetzt n mit einer HTML-Primärfarbe oder einem 6-stelligen RGB-Farbwert. Beispiele: BackgroundGreen, BackgroundFFFF33, BackgroundDefault Wenn die Background-Option nicht verwendet wird (oder BackgroundDefault angegeben ist), dann ist die Hintergrundfarbe die des Fensters oder Tab-Steuerelements.

Range: Bestimmt einen benutzerdefinierten Bereich (standardmäßig von 0 bis 100). Nach dem Wort Range erfolgt der Minimalwert, ein Bindestrich und der Maximalwert. Zum Beispiel: Range0-1000 ermöglicht eine Zahl zwischen 0 und 1000, Range-50-50 eine Zahl zwischen -50 und 50 und Range-10--5 eine Zahl zwischen -10 und -5. In Windows 95 und NT4 verhalten sich negative Bereiche und Bereiche über 65535 erst korrekt, wenn Internet Explorer 3.0 oder höher installiert ist.

-Smooth: Zeigt eine Länge von Segmenten anstelle eines glatten durchgehenden Balkens an. -Smooth ist auch eine Voraussetzung, um einen thematisch gestalteten Fortschrittsbalken in Windows XP oder höher anzuzeigen. Die andere Voraussetzung ist, dass der Balken keine benutzerdefinierten Farben hat, d. h. es wird auf C-und Background-Optionen verzichtet. Windows 95 und NT4 benötigen Internet Explorer 3.0 oder höher, um diese Option zu unterstützen.

Vertical: Der Balken steigt oder fällt, anstatt horizontal bewegend. Windows 95 und NT4 benötigen Internet Explorer 3.0 oder höher, um diese Option zu unterstützen.

Die oben genannten Optionen können mittels GuiControl geändert werden, nachdem das Steuerelement erstellt wurde.

GroupBox

Beschreibung: Ein rechteckiger Rahmen, welcher häufig benutzt wird, um die Gemeinsamkeit von Steuerelementen darzustellen. In diesem Fall enthält der letzte Parameter den Titel des Rahmens, der in der oberen linken Ecke angezeigt wird.

Beispiel: Gui, Add, GroupBox, w400 h300, Geographische Kriterien

Standardmäßig ist der Titel der GroupBox einzeilig. Das kann mit Wrap in den Optionen des Steuerelements überschrieben werden.

Um die Anzahl der Zeilen innerhalb des Steuerelements zu bestimmen (oder die Höhe und Breite), siehe Position und Größe von Steuerelementen anpassen.

Tab2

Beschreibung: Ein großes Steuerelement mit mehreren Seiten, die jeweils weitere Steuerelemente enthalten. Von diesem Punkt an werden diese Seiten "Tabs" genannt.

Tab2 vs. Tab: In v1.0.47.05 wurde das Steuerelement "Tab2" hinzugefügt. Dadurch werden seltene Probleme mit dem Neuzeichnen behoben (ein GUI-Fenster beispielsweise, dass durch Klicken auf bestimmte Teile des Steuerelements wie z. B. Bildlaufleiste aktiviert wird, wird nicht korrekt neu gezeichnet). Das ursprüngliche Tab-Steuerelement wird für die Abwärtskompatibilität beibehalten, weil Tab2 nach seine Steuerelemente in die Navigationsreihenfolge der Tab-Taste hinzugefügt wird. Neue Scripts sollten Tab2 wenn möglich immer verwenden.

Beispiel: Gui, Add, Tab2,, Allgemein|Ansicht|Aussehen|Einstellungen

Der letzte Parameter enthält eine Liste mit Tabnamen, die durch senkrechte Striche dazwischen getrennt werden. Mit zwei senkrechten Strichen danach wird ein Tab vorausgewählt (z. B. Rot|Grün||Blau). Alternativ kann mit dem Wort Choose in den Optionen, unmittelbar gefolgt von der Nummer, ein Eintrag sofort vorausgewählt werden. Mit Choose5 beispielsweise wird der fünfte Tab vorausgewählt (wie bei anderen Optionen, kann es auch eine Variable wie zum Beispiel Choose%Var% sein). Nachdem das Steuerelement erstellt wurde, kann mit GuiControl das ausgewählte Tab geändert oder Tabs hinzugefügt/entfernt werden.

Nachdem das Tab-Steuerelement erstellt wurde, beziehen sich danach hinzugefügte Steuerelemente auf das erste Tab. Das kann jederzeit mit den folgenden Beispielen geändert werden:

Gui, Tab  ; Nachfolgende Steuerelemente gehören nicht dem einem Tab-Steuerelement.
Gui, Tab, 3  ; Nachfolgende Steuerelemente werden dem dritten Tab im Tab-Steuerelement untergeordnet.
Gui, Tab, 3, 2  ; Nachfolgende Steuerelemente werden dem dritten Tab im zweiten Tab-Steuerelement untergeordnet.
Gui, Tab, Name  ; Nachfolgende Steuerelemente werden dem Tab mit der Bezeichnung Name untergeordnet (nicht abhängig von Groß-/Kleinschreibung).
Gui, Tab, Name,, Exact  ; Das Gleiche wie oben, aber mit exakter Übereinstimmung (und abhängig von Groß-/Kleinschreibung).

Es ist auch möglich, Steuerelemente einem Tab oder Tab-Steuerelement zuzuweisen, die noch nicht existieren (außer im Fall der Name-Methode). In diesem Fall aber werden die unten beschriebenen Positionierungsoptionen nicht unterstützt.

Positionierung: Wenn jedes Tab eines Tab-Steuerelements sein erstes Untersteuerelement erhält, dann hat dieses Untersteuerelement eine besondere Standardposition mit folgenden Bedingungen: 1) Sowohl die X-als auch Y-Koordinaten sind nicht vorhanden, in diesem Fall wird das erste Untersteuerelement in der oberen linken Ecke des Tab-Steuerelements positioniert (mit Standard-Abstand) und jeder Untersteuerelement über dem Ersten wird unter dem vorherigen Steuerelement positioniert; 2) Die Positionierungsoptionen X+n und/oder Y+n wurden angegeben, in diesem Fall beziehen sich die Untersteuerelemente beim Positionieren auf die linke obere Ecke des Tab-Steuerelements. Mit "x+10 y+10" wird das Steuerelement von der linken oberen Ecke 10 Pixel nach rechts und 10 Pixel nach unten positioniert.

Untersteuerelemente müssen nicht unbedingt in der Abgrenzung des Tab-Steuerelements vorhanden sein: sie werden weiterhin unsichtbar und sichtbar gemacht, sobald ihr Tab ausgewählt oder abgewählt wird. Dieses Verhalten ist besonders für den unten beschriebenen Schaltflächen-Style geeignet.

Wenn der Befehl Gui Submit verwendet wird, dann wird mit der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) der Name des zurzeit ausgewählten Tabs abgerufen. Wenn jedoch das Steuerelement die AltSubmit-Eigenschaft hat, dann wird mit der Ausgabevariable stattdessen die Positionsnummer des Tabs abgerufen (das erste Tab ist 1, das Zweite 2 usw.).

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer zu einem neuen Tab wechselt. Wenn das Tab-Steuerelement sowohl ein g-Label als auch eine Ausgabevariable hat, dann enthält die Ausgabevariable den vorher ausgewählten Tabnamen (oder Nummer im Falle von AltSubmit), sobald der Benutzer auf ein neues Tab wechselt.

Navigation mit der Tastatur: Der Benutzer kann mithilfe von Steuerung+Bild-Ab/Bild-Auf von Tab zu Tab navigieren; sobald ein Steuerelement den Tastaturfokus hat, das nicht zum Tab-Steuerelement gehört, dann wird das erste Tab-Steuerelement des Fensters angesteuert. Steuerung+Tab und Steuerung+Umschalt+Tab können auch verwendet werden, mit der Ausnahme, dass es nicht funktioniert, wenn das fokussierte Steuerelement ein mehrzeiliges Edit-Steuerelement ist.

Jedes Fenster kann nicht mehr als 255 Tab-Steuerelemente haben. Jedes Tab-Steuerelement kann nicht mehr als 256 Tabs (Seiten) haben. Darüber hinaus kann ein Tab-Steuerelement keine anderen Tab-Steuerelemente enthalten.

Tab-Optionen:

Choose: Siehe oben.

-Background: Überschreibt die benutzerdefinierte Hintergrundfarbe des Fensters und verwendet die Standardfarbe des Betriebssystems. Mit +Theme -Background wird das Tab-Steuerelement an das aktuelle Desktop-Design angepasst. Allerdings werden die meisten Steuerelementarten seltsam damit im Tab-Steuerlement aussehen, weil ihre Hintergründe nicht mit dem Hintergrund des Tab-Steuerelements übereinstimmen. Bei einigen Steuerelementarten (z. B. Text) kann das Problem mit BackgroundTrans in deren Optionen umgegangen werden.

Buttons: Erstellt mehrere Schaltflächen anstelle von Tabs am oberen Rand des Steuerelements (in diesem Fall ist standardmäßig kein Rahmen vorhanden, weil die Anzeigefläche der Regel keine Steuerelemente enthält).

Left/Right/Bottom: Mit diesen Wörtern werden die Tabs an der linken, rechten oder unteren anstelle der oberen Seite angezeigt. Siehe TCS_VERTICAL für Beschränkungen bei links und rechts.

-Wrap: Verhindert, dass die Tabs mehr als eine Zeile verbrauchen (in diesem Fall werden Pfeil-Schaltflächen angezeigt, wenn es zu viele Tabs sind und erlaubt dem Benutzer, alle Tabs einzusehen).

Um die Anzahl der Zeilen innerhalb des Steuerelements zu bestimmen (oder die Höhe und Breite), siehe Position und Größe von Steuerelementen anpassen.

Icons in Tabs: Mittels SendMessage kann ein Icon neben dem Namen des Tabs angezeigt werden. Das wird im Forumsbeitrag Icons in tabs demonstriert.

StatusBar[v1.0.44+]

Beschreibung: Eine Reihe mit Text und/oder Icons am unteren Rand des Fensters, die typischerweise verwendet wird, um geänderte Zustände zu melden.

; Beispiel:
Gui, Add, StatusBar,, Starttext der Leiste (kann leer gelassen werden).
SB_SetText("Es sind " . RowCount . " Zeilen markiert.")

Eine Statusleiste kann am einfachsten verwendet werden, indem SB_SetText() aufgerufen wird, sobald sich etwas ändert, das dem Benutzer gemeldet werden soll. Um mehrere Informationen zu melden, teilt die Leiste mittels SB_SetParts() in mehrere Abschnitte. Mit SB_SetIcon() können Icons in der Leiste angezeigt werden.

Alle folgenden StatusBar-Funktionen laufen im Standard-GUI-Fenster des aktuellen Threads (kann mittels Gui, 2:Default geändert werden). Wenn das Standard-Fenster weder vorhanden ist noch eine Statusleiste hat, dann geben alle SB-Funktionen eine 0 zurück, um auf ein Problem hinzuweisen.

SB_SetText(NewText [, PartNumber, Style]): Zeigt NewText im angegebenen Teil der Statusleiste an. Wenn PartNumber nicht vorhanden ist, dann ist 1 der Standardwert. Andernfalls kann ein Integer-Wert zwischen 1 und 256 angegeben werden. Wenn Style nicht vorhanden ist, dann ist 0 der Standardwert, damit wird der traditionelle Rahmen verwendet, d. h. der Abschnitt sieht aus wie eingefallen. Andernfalls kann eine 1 angegeben werden, um den Rahmen zu entfernen oder eine 2, um einen Rahmen zu haben, damit der Abschnitt der Leiste angehoben wirkt. Schließlich können bis zu zwei Tab-Zeichen (`t) in NewText angegeben werden: alles auf der rechten Seite des ersten Tabs ist zentriert und alles auf der rechten Seite des zweiten Tabs ist rechtsbündig. SB_SetText() gibt bei Erfolg eine 1 und bei Misserfolg eine 0 zurück.

SB_SetParts([Width1, Width2, ... Width255]): Teilt die Leiste in mehrere Abschnitte, abhängig von der angegebenen Breite (in Pixel). Wenn alle Parameter weggelassen werden, dann hat die Leiste nur einen langen Abschnitt. Andernfalls kann die Breite für jeden Abschnitt angegeben werden (der letzte Abschnitt hat die verbleibende Breite der Leiste). Mit SB_SetParts(50, 50) werden drei Abschnitte erstellt: die ersten zwei Abschnitte haben eine Breite von 50 und der Letzte die verbleibende Breite. Hinweis: Jeder Abschnitt, der mit SB_SetParts() "gelöscht" wurde, startet beim nächsten Mal ohne Text, wenn er angezeigt wird (außerdem werden die Icons automatisch zerstört). Bei Erfolg gibt SB_SetParts() einen Wert größer 0 zurück (HWND der Statusleiste). Bei Misserfolg eine 0.

SB_SetIcon(Dateiname [, IconNumber, PartNumber]): Zeigt ein kleines Icon auf der linken Seite des Textes im angegebenen Abschnitt an (wenn PartNumber weggelassen wird, dann ist 1 der Standardwert). Der Dateiname enthält den Namen einer Icon-Datei (.ICO), einer Cursor-Datei (.CUR) oder einer animierten Cursor-Datei (.ANI) (animierte Cursor werden zurzeit nicht in der Leiste animiert). Andere Quellen von Icons können sein: EXE, DLL, CPL, SCR und andere Typen, die Icon-Ressourcen enthalten. Um eine beliebige Icon-Gruppe in einer Datei zu verwenden, verwendet die Gruppennummer in der Iconnummer. Mit SB_SetIcon("Shell32.dll", 2) beispielsweise wird das Standard-Icon aus der zweiten Icon-Gruppe verwendet. SB_SetIcon() gibt bei Erfolg HICON des Icons und bei Misserfolg eine 0 zurück. HICON ist eine System-Ressource, die von den meisten Scripts getrost ignoriert werden kann, weil es automatisch zerstört wird, wenn das Fenster der Statusleiste zerstört wird. Ebenso wird jedes alte Icon zerstört, wenn es mit einem neuen Icon via SB_SetIcon() ersetzt wird. Das kann wie folgt verhindert werden:

Gui +LastFound
SendMessage, 0x40F, part_number - 1, eigenes_hicon, msctls_statusbar321  ; 0x40F ist SB_SETICON.

SB_SetProgress(): Erstellt und steuert einen Fortschrittsbalken innerhalb einer Statusleiste. Diese Funktion ist auf www.autohotkey.com/forum/topic37754.html verfügbar.

g-Label-Meldungen: Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Damit wird EigeneSubroutine automatisch ausgeführt, sobald der Benutzer auf die Leiste klickt. Diese Subroutine kann auf die internen Variablen A_Gui und A_GuiControl zurückgreifen. Noch wichtiger ist, dass sie auf die interne Variable A_GuiEvent zurückgreifen kann, die einen folgenden String enthalten kann (aus Kompatibilitätsgründen mit zukünftigen Versionen sollte ein Script nicht davon ausgehen, dass das die einzigen möglichen Werte sind):

  • Normal: Der Benutzer hat mit der linken Maustaste auf die Leiste geklickt. Die interne Variable A_EventInfo enthält die Abschnittsnummer (kann aber auch ein sehr großer Integer-Wert sein, wenn der Benutzer auf den Größenziehpunkt in der rechten Seite der Leiste klickt).
  • RightClick: Der Benutzer hat mit der rechten Maustaste auf die Leiste geklickt. Die interne Variable A_EventInfo enthält die Abschnittsnummer. HINWEIS: GuiContextMenu wird nicht für die Statusleiste aufgerufen, wenn ein g-Label vorhanden. Auch das RightClick-Ereignis des g-Labels sollte anstelle von GuiContextMenu verwendet werden, wenn das Script wissen muss, auf welcher Abschnittsnummer der Benutzer geklickt hat (A_EventInfo).
  • DoubleClick: Der Benutzer hat mit der linken Maustaste doppelt auf die Leiste geklickt. Die interne Variable A_EventInfo enthält die Abschnittsnummer.
  • R: Der Benutzer hat mit der rechten Maustaste doppelt auf die Leiste geklickt. Die interne Variable A_EventInfo enthält die Abschnittsnummer.

Schrift und Farbe: Obwohl die Größe, Art und Stil der Schrift mittels Gui Font eingestellt wird (wie bei einem normalen Steuerelement), kann die Textfarbe nicht geändert werden. Auch "Gui Color" wird nicht berücksichtigt; Stattdessen kann die Hintergrundfarbe der Statusleiste mit dem Wort Background in den Optionen geändert werden, gefolgt vom Farbnamen (siehe Farbtabelle) oder RGB-Wert (0x-Präfix ist optional). Beispiele: BackgroundSilver, BackgroundFFDD99, BackgroundDefault

Ausblenden der Statusleiste: Die Leiste kann bei der Erstellung mittels Gui, Add, StatusBar, Hidden vEigeneStatusBar versteckt werden. Nach der Erstellung kann sie mit GuiControl, Hide, EigeneStatusBar versteckt werden. Um sie anzuzeigen, verwendet GuiControl, Show, EigeneStatusBar. Hinweis: Das Ausblenden der Leiste reduziert nicht die Höhe des Fensters. Wenn das gewünscht ist, einfach Gui, Show, AutoSize verwenden.

Styles (selten verwendet): Siehe Style-Tabelle der StatusBar.

Bekannte Einschränkungen: 1) Jedes Steuerelement, dass die Statusleiste überdeckt, wird manchmal drüber gezeichnet. Zur Vermeidung dieses Problems können solche Steuerelemente mittels GuiSize label dynamisch geschrumpft werden. 2) Jedes Fenster kann nur eine Statusleiste besitzen.

Beispiel: Im unteren Bereich der TreeView-Seite wird eine Statusleiste mit mehreren Abschnitten demonstriert.
