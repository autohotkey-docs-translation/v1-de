#MenuMaskKey [AHK_L 38+]

Ändert die Taste, die dazu verwendet wird, Win- oder Alt-Up-Ereignisse zu verbergen.

#MenuMaskKey Tastenname

Parameter

Tastenname Ein Tastenname oder eine vkNN-Sequenz, die einen virtuellen Tastencode ungleich 0 bestimmt. Scan-Codes können nicht verwendet werden.

Bemerkungen

Diese Einstellung ist global - das heißt, dass diese Direktive nur einmal im Script angegeben werden muss (egal wo), um das Verhalten des ganzen Scripts zu beeinflussen.

Falls ein Hotkey mithilfe des Tastatur- oder Maus-Hooks implementiert worden ist, könnte die letzte Tastatureingabe für das aktive Fenster und System unsichtbar sein. Bei Hotkeys, die die Win- oder Alt-Modifikatortaste verwenden, könnte das dazu führen, dass das Startmenü oder die Menüleiste des aktiven Fensters aktiviert wird, sobald die Modifikatortaste losgelassen wird. Damit das nicht passiert, "versteckt" AutoHotkey das KeyUp-Ereignis durch Senden einer Tastatureingabe. Vor Revision 38 war das immer eine Strg-Tastatureingabe, das aber bei bestimmten Anwendungen zu Problemen führte.

Wenn diese Direktive nicht angegeben ist, gilt standardmäßig die Strg-Taste.

Verwandte Befehle

Siehe diesen Thread für Hintergrundinformationen.

Beispiele

#MenuMaskKey vk07  ; vk07 ist unbelegt.
#UseHook
#Space::
!Space::
    KeyWait LWin
    KeyWait RWin
    KeyWait Alt
    KeyHistory
