AutoIt-v2-Kompatibilität

In AutoHotkey v1.1.09 und höher werden AutoIt-v2-Scripts (.aut) nicht länger unterstützt. Ältere Versionen von AutoHotkey unterstützen AutoIt-v2-Scripts, indem die Standardeinstellungen und Verhalten einiger Befehle geändert werden. Diese Seite enthält Informationen, die auf den anderen Seiten der Dokumentation entfernt wurden.

Folgendes gilt nur für .aut-Dateien, und nur bei v1.1.08.01 oder niedriger.

#AllowSameLineCommentsStandardmäßig dürfen Kommentare nicht neben einen Befehl vorkommen.
A_ScriptDirEin letzter Backslash ist enthalten.
DetectHiddenTextStandardeinstellung ist OFF.
#EscapeCharStandardeinstellung ist Backslash (\).
FileCopyErrorLevel wird auf 1 gesetzt, falls eine Datei nicht kopiert werden konnte.
IniDeleteErrorLevel bleibt unverändert.
IniReadDer Standardwert-Parameter wird nicht unterstützt; der String ERROR wird immer in AusgabeVar gespeichert, falls der Wert nicht gelesen werden kann.
IniWriteErrorLevel bleibt unverändert.
InputBoxWenn der Benutzer die Schaltfläche CANCEL drückt, wird AusgabeVar leer gemacht. ErrorLevel bleibt unverändert, solange das Dialogfenster die Zeit nicht überschreitet.
SetBatchLinesStandardeinstellung ist 1, wodurch das Script nach jeder Zeile eine Pause macht.
SetKeyDelayStandardeinstellung für Druckdauer beim traditionellen SendEvent-Modus ist 20.
SendDas Zeichen # wird ignoriert.
SplashTextOnHöhe schließt die Titelleiste des Fensters mit ein.

Escapezeichenumwandlung

Das Starten einer Script-Datei mit der Erweiterung .aut.ahk bei AutoHotkey v1.1.08.01 oder niedriger führt das Script nicht aus; stattdessen wird das Standardescapezeichen von AutoIt v2 (Backslash) in das Standardescapezeichen von AutoHotkey (Backtick) umgewandelt.

Veraltete Befehle

Die folgenden Befehle wurden automatisch in den entsprechenden AutoHotkey-Befehlen übersetzt (das zuvor nicht dokumentiert war): LeftClick, LeftClickDrag, RightClick, RightClickDrag, HideAutoItWin, Repeat, EndRepeat. Diese automatische Übersetzung wurde in v1.1.09 entfernt.
